# 学习工具与课程资料的信息架构审视

## 审视范围

当前学习页底部的“学习”“资源”两个入口，以及它们所包含的 5 个二级项目：

- 学习：当前练习、学习记录、学习概况
- 资源：知识库、教学资源

## 总体判断

现在的设计把两个“分类”做成了两个“页面入口”。用户看见的是“学习 / 资源”两个并列按钮，但看不见它们下面有什么；点击后又会直接跳到上一次使用的子项目，因此入口含义和点击结果都不够稳定。

这不是配色或按钮样式的问题，而是层级没有显性表达。

## 建议的信息架构

把“学习”和“资源”从可切换的一级页面，改成一个“学习工具”面板中的两个分组：

```text
学习工具
├─ 学习任务（做什么）
│  ├─ 当前练习
│  ├─ 学习记录
│  └─ 学习概况
└─ 课程资料（查什么）
   ├─ 知识库
   └─ 教学资源
```

底部导航建议保留两个明确入口：

```text
[ 学习工具 · 5 ]   [ 智能助教 ]
```

点击“学习工具”后，在桌面端打开锚定在按钮上方的面板，在移动端打开底部抽屉。面板同时展示两个分组和全部 5 个项目，不再让用户先猜分类、再寻找功能。

## 面板文案与结构

```text
学习工具                                      ×
围绕当前章节练习、记录学习过程或查看课程资料

学习任务 · 3
┌ 当前练习 ── 完成当前章节的正式练习 ── 继续/未开始 ┐
├ 学习记录 ── 查看笔记、问答和待复习内容 ── 3 条 ───┤
└ 学习概况 ── 查看阅读进度、掌握情况和学习证据 ────┘

课程资料 · 2
┌ 知识库 ──── 查看本课知识结构与课程覆盖 ──────────┐
└ 教学资源 ── 查看大纲、教案、讲义、练习册和演示 ─┘
```

### 命名调整

- “学习”改为“学习任务”：说明这里是用户要做的事，不与整个学习页面重名。
- “资源”改为“课程资料”：缩小含义范围，避免被理解成下载、文件或外部链接。
- 总入口使用“学习工具”：它是一个功能集合，不用选中态伪装成当前页面。

## 交互规则

1. 点击总入口只打开工具面板，不自动跳转到上次使用的子项目。
2. 点击具体项目后再进入对应覆盖层。
3. 在子项目覆盖层顶部保留面包屑，例如“学习工具 / 学习任务 / 学习记录”，并提供“全部工具”返回入口。
4. 当前项目使用明确的行级高亮；底部总入口只在面板或子项目打开时显示激活态。
5. “当前练习”不可用时不要只置灰，同时显示原因，例如“本节暂无正式练习”。
6. 智能助教保持独立，因为它是贯穿正文、练习和资料的上下文助手，不属于上述任一分组。

## 视觉与可访问性注意

- 分组标题、项目标题和说明至少形成三级清晰层级。
- 子项目整行可点击，桌面端和移动端都保持足够的点击区域。
- 总入口应使用 `aria-haspopup="dialog"` 和 `aria-expanded`；两个分组使用语义标题和列表，不再把分组按钮标记为 `aria-current="page"`。
- 置灰项目必须同时提供可见原因，不能只依赖悬浮提示。
- 截图只能确认视觉层级问题；键盘焦点、焦点回退、读屏顺序和缩放重排仍需在实际页面验证。

## 审视步骤

1. **学习页正文与底部入口：需改进。** 两个分类表现成页面切换，5 个子项目完全隐藏。
2. **进入“学习”后的二级项目：需改进。** 三个同级功能只在覆盖层内部出现，用户进入前无法预判。
3. **进入“资源”后的二级项目：需改进。** “资源”命名过宽，且知识库与教学资源的差异没有在入口处解释。
4. **智能助教：健康。** 作为跨场景助手独立放置是合理的，不建议并入 5 个工具。
