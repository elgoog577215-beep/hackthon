# 演示触发路径（录屏依据）

对应需求：`docs/requirements/2026-07-19-demo-video-requirements.md`。
两条视频围绕同一门《矩阵与线性变换》展开。正式产品仍支持从课程需求生成全课；
实际录屏为减少等待和无效动作，统一使用手工定稿、可重复重置的专用课程快照。
快照只冻结录制起点，之后的语义理解、影响分析、用户确认、课程修订、教学材料同步、
个体化内容生长和效果复验全部走真实产品链路。

正式录制固定使用完整 `16:9` 页面和浏览器 `100%` 缩放，不做画面放大、缩小、追焦
或镜头平移。详细时间线以
[启智大会双视频正式录制稿](./requirements/2026-07-21-recordly-demo-storyboards.md)
为准。

## 双视频录制环境准备

macOS / Linux：

```bash
./dev.sh --recordly-demo
```

Windows：先运行 `python scripts/prepare_recordly_demo.py`，再设置
`EVOLUTION_DEMO_MODE=1`、`VITE_RECORDLY_DEMO_MODE=1` 与
`VITE_LEARNER_USER_ID=video2-demo-student` 后启动 `dev.bat`。

该命令只在服务启动前重置录屏专用课程 `demo-matrix-growth-v2` 及其个人学习状态，不修改
其他课程。演示模式会放宽强证据触发，并让 AI 老师使用本地定稿回答，因此没有
API Key、模型配额或网络波动也能完成录制；不设置时仍走生产阈值与真实模型。
专用课程本身为手工定稿的 4 章 12 节完整微型课程，覆盖向量、矩阵、复合变换、
基变换、逆变换、图形管线、特征方向、动态系统、投影、最小二乘与综合项目，
并为每节准备正式练习。基线课程保留“会算但不理解复合顺序”的可生长缺口，
不会提前写入视频二确认后才出现的几何动画与针对性练习。

视频二直达地址：
<http://localhost:5174/course/demo-matrix-growth-v2/learn/v2-sec-1-2>

视频一直达地址：
<http://localhost:5174/course/demo-matrix-growth-v2/ppt>

重新录制时先按 `Ctrl+C` 停止服务，再重新执行 `./dev.sh --recordly-demo`。不得在
服务运行中直接执行重置脚本，否则运行进程可能继续持有旧缓存。

## 视频一可选前置：现场生成新课程

这段仅在需要证明“从一句需求生成整门新课”时使用，可单独剪成过场；大会主视频默认
使用上面的确定性起点，把镜头时间留给教学材料生成结果和联动更新闭环。

1. 打开 `/courses`，点击右上角「新建课程」。
2. 在「课程主题」填写 `矩阵与线性变换`，选择「进阶」「智能均衡」「自动匹配」和
   「系统学习」；资料使用边界选择「资料与通用知识结合」。
3. 在「额外要求」填写：

   > 生成 4 章、每章 3 节，共 12 节。覆盖向量与矩阵、矩阵乘法、线性变换、复合变换、
   > 特征值的直观含义；每节给出明确目标、核心讲解、理解检查和可追溯练习。“矩阵乘法”
   > 小节的学习目标写成“掌握矩阵乘法的计算规则”。

4. 点击「确认需求，生成目录」。页面应直接进入同一门课程的学习现场；不要返回课程库
   再创建第二门课。记录地址栏中的 `courseId`，后续 PPT 和学习视频均使用此 ID。
5. 等待「检查并确认课程目录」。核对目录为 4 章 12 节且包含“矩阵乘法/复合变换”；
   将“矩阵乘法”小节目标校对为「掌握矩阵乘法的计算规则」，再点击「确认目录并继续」。
   若结构不符，停止录制并在目录确认前修正或重新创建，不能拿另一门旧课替代。
6. 在生成现场观察真实阶段：冻结全课知识职责 → 分批规划并汇编小节教案 → 编译知识库
   与关系图 → 逐节正文 → 练习与发布预检。期间可打开「生成任务」查看批次、已完成小节
   和正文数量；百分比只表示真实检查点，不口播“内容已完成多少”。
7. 失败时在「生成任务」点击继续/恢复，确认提示为复用已保存检查点，并保持原
   `courseId`；不要删除现场后新建课程。恢复后已完成批次和正文不应重做。
8. 到「课程已经长成，可以确认发布」时点击「确认并发布」。发布完成后同一 URL 原地
   成为正式学习页。确认课程标题、4 章 12 节目录和正文可见，再开始视频一。

录屏前清理浏览器下载目录中的同名 `.pptx`，并保持浏览器缩放 100%，避免导出验收时
误开旧文件。

## 视频一：结构化同源（PPT 工作台）

1. **教学材料一键生成结果**：打开预置 PPT 路由，点击右上角「教学材料总览」。
   首屏同时呈现大纲、教案、讲义、PPT、练习册和知识图解六类材料均与同一课程结构
   连接；逐类点开 3–5 秒，证明不是多个互不相干的文件。
2. **进入 PPT 更新**：关闭总览，状态应为「质量检查通过」；右侧「同源依据」显示
   知识、能力或正式题目绑定。
3. **修改学习目标**：左侧缩略图选中“学习目标”页；右侧检查器「页面修改」区
   已自动定位到 `key_message` 字段。把
   「掌握矩阵乘法的计算规则」改为
   「理解矩阵乘法为什么表示线性变换的复合，并能解释运算顺序」。
4. **分析影响**：点击「分析影响」按钮（输入≥8 字后 900ms 也会自动预览）。
   - 会看到：语义识别卡显示教学维度跃迁 **计算技能 → 概念理解**、
     解读文案（"这不只是措辞调整……"）与教学落点清单。
5. **影响范围弹窗**：受影响项（教案重点/正文/图解/例题/理解检查）以**黄色**
   卡片列出（含联动理由）；下方「不该动的，明确保持不变」区列出受保护项与数量；
   底部统计「N 处预计联动 / M 处保持不变」。
6. **生成课程修改候选**：点击「改变课程含义并联动」。此时课程真源尚未改变；弹窗应
   显示修改前后、影响理由和待同步范围，不能直接跳到成功态。
7. **确认精准同步**：点击「确认并同步相关内容」。等待真实课程修订和材料重编译完成。
   回执分别显示「实际更新」「仅校验」和「确认无需处理」数量；使用结果导览依次点击
   大纲、教案、讲义、演示文稿和练习册，每类停留 3–5 秒，右侧必须出现真实修改前后
   或来源校验结果。数量以本次真实结果为准，不在口播中写死。
8. **导出并验收**：点击右上角「导出 PPTX」。确认下载的是当前课程标题命名的新文件；
   用原生 PowerPoint 打开，核对页数与工作台一致、中文和公式无乱码、文本框可编辑、
   讲者备注存在。
9. **落版镜头**：精准同步回执 + 保持不变区 + PowerPoint 中可编辑对象。字幕使用
   “可在 PowerPoint 继续使用；同源语义修改回到灵知工作台完成”，不要宣称支持外部
   PPTX 回导。

### 可选故障镜头：上一可用课件不被覆盖

这段只用于故障兜底验收，不建议塞入 55–65 秒主片。

1. 先完成一次成功课件构建，刷新页面并确认正式页仍在。
2. 打开浏览器开发者工具，在 Network 中切换为 `Offline`，点击「同步课程」。
3. 请求失败后应看到「本次生成失败，当前展示上一可用版本」，原正式页序和内容仍在；
   不应出现空白成功态，也不应把临时问题页当成正式课件。
4. 恢复 `No throttling`，再次点击「同步课程」。成功后故障提示消失。随后再执行导出，
   避免把“网络离线导致下载失败”误判为 PPTX 生成问题。

## 视频二：个体化生长（学习工作区）

1. **进入**：打开《矩阵与线性变换》→ 学习工作区，定位到
   「1.2 矩阵：线性映射与矩阵运算」。直达路由为
   `/course/demo-matrix-growth-v2/learn/v2-sec-1-2`，左侧课程目录全程可见。
2. **提问**：右侧打开 **AI 老师** 面板，输入固定演示文案：
   > 矩阵乘法计算我会，但我一直不理解为什么复合变换要先右后左。请在本节和
   > 后面相关内容中，先用几何动画解释，再让我进行计算。
3. **发送**：提问事件自动触发生长评估。课程生长面板出现
   「**已识别强学习证据**」卡：
   - 四个语义维度逐条显示解析出的原文：已会什么（计算我会）/ 卡在哪里
     （不理解为什么复合变换要先右…）/ 希望怎样讲（分步解释、几何动画、
     再做计算）/ 调整到哪里（本节及相关后续）。
4. **方案预览**：生长方案卡展开：当前位置解释、分步动画、针对性练习、
   后续过渡、理解检查（growth-trail 三步在目录树下也可见）。
   左侧目录：本小节与后续相关节点出现**紫色「AI 建议」**标记；仅在影响
   范围内、无直接修改的节点显示**黄色「受影响」**。
5. **范围选择**：方案卡内可切换「只应用本小节」/「本小节及后续」。
6. **确认应用**：点击确认 → 左侧目录标记先闪烁并转为**蓝色「已应用」**
   （不是绿色！）→ 正文自动跳转到本次新增内容 → 新增解释/动画块高亮闪烁后稳定；
   课程生成新版本，方案卡提供「撤销」。
7. **播放动画**：滚动到新增的分步动画块，点击播放：SVG 依次展示
   `v → Bv → A(Bv)`（网格/向量随关键帧变换），含"先 B 再 A"交互判断题。
8. **闭合效果复验**：在动画判断中选择“先 B 再 A”，再进入新增的针对性正式练习，
   选择正确状态链并提交。该题由确定性合同评分，不依赖外部模型；课程生长面板刷新为
   绿色「本轮独立复验通过」。
9. 落版镜头：目录蓝色「已应用」+ 面板绿色「本轮独立复验通过」，同时保留“仍需后续
   不同任务持续确认”的诚实边界。

## 四色状态速查（两条视频共用）

- 蓝：当前正式课程 / 已应用未复验
- 黄：受影响位置（影响预览卡、目录「受影响」标记）
- 紫：AI 待确认方案（目录「AI 建议」、方案卡）
- 绿：**仅**复验通过（`effect_evaluation = effective` 的「已生长」）
