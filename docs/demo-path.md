# 演示触发路径（录屏依据）

对应需求：`docs/requirements/2026-07-19-demo-video-requirements.md`。
两条视频共用同一门新生成的《矩阵与线性变换》课程。

启动：项目根目录 `.\dev.bat`（前端 <http://localhost:5173>，后端 8000）。
录视频二前，后端需以 `EVOLUTION_DEMO_MODE=1` 环境变量启动（放宽强证据触发，
仅演示用；不设置时走生产阈值）。

## 视频一：结构化同源（PPT 工作台）

1. **进入**：课程库 → 打开《矩阵与线性变换》→ 顶部进入 **PPT 工作台**
   （路由 `/course/{courseId}/ppt`）。若是空态，点「生成完整课件」等待编译完成。
2. **修改学习目标**：左侧缩略图选中"学习目标"页；右侧检查器「页面修改」区
   已自动定位到 `key_message` 字段。把
   「掌握矩阵乘法的计算规则」改为
   「理解矩阵乘法为什么表示线性变换的复合，并能解释运算顺序」。
3. **分析影响**：点击「分析影响」按钮（输入≥8 字后 900ms 也会自动预览）。
   - 会看到：语义识别卡显示教学维度跃迁 **计算技能 → 概念理解**、
     解读文案（"这不只是措辞调整……"）与教学落点清单。
4. **影响范围弹窗**：受影响项（教案重点/正文/图解/例题/理解检查）以**黄色**
   卡片列出（含联动理由）；下方「不该动的，明确保持不变」区列出受保护项与数量；
   底部统计「N 处预计联动 / M 处保持不变」。
5. **确认联动**：选择「课程语义修改」路径确认 → 系统回写课程目标真源并生成
   授权变更 → 点击确认联动。受影响卡片依次转绿，底部出现
   「**N 项实际更新 / M 项确认无需改写**」回执（另有"仅校验"档）。
6. 落版镜头：回执统计 + 保持不变区。

## 视频二：个体化生长（学习工作区）

1. **进入**：打开同一门课 → 学习工作区（`/course/{courseId}/learn/...`），
   左侧课程目录全程可见。定位到「矩阵乘法」相关小节。
2. **提问**：右侧打开 **AI 老师** 面板，输入固定演示文案：
   > 矩阵乘法计算我会，但我一直不理解为什么复合变换要先右后左。请在本节和
   > 后面相关内容中，先用几何动画解释，再让我进行计算。
3. **发送**：提问事件自动触发生长评估。课程生长面板出现
   「**已识别强学习证据**」卡：
   - 四个语义维度逐条显示解析出的原文：已会什么（计算我会）/ 卡在哪里
     （不理解为什么复合变换要先右…）/ 希望怎样讲（分步解释、几何动画、
     再做计算）/ 调整到哪里（本节及相关后续）。
4. **方案预览**：生长方案卡展开：当前位置解释、分步动画、针对性练习、
   后续过渡、理解检查（growth-trail 三步在目录树下也可见）。
   左侧目录：本小节与后续相关节点出现**紫色「AI 建议」**标记；仅在影响
   范围内、无直接修改的节点显示**黄色「受影响」**。
5. **范围选择**：方案卡内可切换「只应用本小节」/「本小节及后续」。
6. **确认应用**：点击确认 → 左侧目录标记转**蓝色「已应用」**（不是绿色！），
   课程生成新版本；方案卡提供「撤销」。
7. **播放动画**：滚动到新增的分步动画块，点击播放：SVG 依次展示
   `v → Bv → A(Bv)`（网格/向量随关键帧变换），含"先 B 再 A"交互判断题。
8. 落版镜头：目录蓝色「已应用」+「等待独立复验」状态（绿色只在后续正式
   证据通过后出现）。

## 四色状态速查（两条视频共用）

- 蓝：当前正式课程 / 已应用未复验
- 黄：受影响位置（影响预览卡、目录「受影响」标记）
- 紫：AI 待确认方案（目录「AI 建议」、方案卡）
- 绿：**仅**复验通过（`effect_evaluation = effective` 的「已生长」）
