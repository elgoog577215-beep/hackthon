# 课程生成到学习闭环全链路验收矩阵

## 1. 状态与目标

- 所属全景：课程使用第 8 步“全链路验收与回看课程生成”。
- 当前状态：8.1 验收矩阵、8.2 只读预检、8.3 生成契约与真实标准样本、8.4 主学习链、8.5A 前端主链组装、8.5B 版本变化和 8.5C 降级验收均已完成；当前进入 8.6 缺陷回灌，尚未宣称第 8 步整体完成。
- 核心目标：证明一门课程不是“生成后能打开”，而是能够连续学习、可靠恢复、正式验证、诊断补救、解释下一步，并在课程版本变化后保留可说明的历史。

本轮不新增第八套业务状态。验收只读取课程、版本、学习快照、学习记录、正式作答、诊断补救、学习运行时和 AI 老师协议各自的正式真源。

## 2. 固定验收样本

### 2.1 标准全链路样本

必须使用统一课程生成链新生成一门 `systematic + math_formal` 课程，至少包含两个章节、每章两个目标，并满足：

- 有明确课程版本和生成任务记录。
- 每个 L2 目标有稳定目标修订和至少两个正文内容块。
- 每个目标有理解检查、目标练习、掌握检查和对应量规。
- 至少一个目标有通用误区、辨别任务、最小补救单元和独立复验题。
- 至少存在一个显式可靠前置关系，可验证章节进入与收束。

标准样本不得通过测试代码临时注入正式题目，也不得在验收时现场拼接缺失资产。生成器没有产出的内容应直接记为课程生成缺陷。

### 2.2 旧课程兼容样本

固定使用课程 `9f0727b7-aba9-4776-ab2e-3fe03a058472`（线性代数入门 smoke test）及其历史版本验证：

- 旧正文能投影稳定内容块和目标修订。
- 缺少正式练习时明确显示证据不足，不伪造掌握。
- 版本变化后快照、记录和历史作答进入可解释的迁移或失效状态。

该课程当前正式练习资产为空，只能作为兼容与降级样本，不能作为标准 happy path。

### 2.3 降级样本

阅读型课程使用统一资产编译器生成的 `material_organization` 契约夹具验证。它按生成契约不提供正式题目，并在课程计划、前端状态和学习运行时中明确标记为阅读型降级。当前仓库没有持久化的真实阅读型课程，因此不能拿旧课程无题状态冒充该样本。

### 2.4 验收身份与隔离

- 设备 A：`acceptance-main-device-a`
- 设备 B：`acceptance-main-device-b`，使用同一用户身份模拟跨设备。
- 对照用户：`acceptance-control-user`
- 验收产生的记录、Attempt、诊断、会话和回执必须按用户与课程隔离；不得使用默认用户或已有真实学习数据。

## 3. 六项课程使用契约

| 契约 | 课程生成必须提供 | 课程使用必须证明 |
| --- | --- | --- |
| 版本契约 | 课程 ID、版本 ID、生成任务与资产修订 | 刷新和版本变化后能区分当前对象与历史对象 |
| 目标契约 | 稳定目标 ID、目标修订、完成与掌握标准 | 阅读完成和掌握证据互不冒充 |
| 内容契约 | 稳定内容块、指纹、修订和语义位置 | 快照、笔记和 AI 引用可以恢复或解释迁移 |
| 练习契约 | 正式任务修订、用途、量规、答案规范 | 草稿可恢复、提交幂等、支持等级影响证据 |
| 补救契约 | 误区、辨别任务、补救单元和独立复验 | 错因先验证，补救后必须用新题独立复验 |
| 推进契约 | 章节目标、前置关系和完成策略 | 首次进入、继续学习和章节结束只有一个主行动 |

AI 老师是六项契约的消费者和动作协调者，不是第七份课程资产，也不拥有独立学习状态。

## 5. 失败归属与严重级别

### 5.1 失败归属

- `GEN`：生成任务、课程版本、课程结构或发布过程错误。
- `ASSET`：目标、内容块、正式题目、量规、误区或补救资产缺失和错绑。
- `DOMAIN`：快照、记录、Attempt、诊断或版本迁移的领域状态机错误。
- `RUNTIME`：多个领域都正确，但聚合修订、活动任务或唯一下一步错误。
- `AI`：上下文、答案隔离、提案、回执、拒绝冷却或动作权限错误。
- `FE`：后端状态正确，但页面展示、恢复、交互、国际化或响应式错误。
- `ENV`：模型凭据、网络、端口或外部服务导致的环境失败。

一个问题只能有一个主归属，可附带协同归属。不能用“前后端都有问题”代替定位。

### 5.2 严重级别

- `P0`：数据串用户、正式答案泄露、状态丢失、重复提交、错误掌握或不可逆历史破坏。立即停止整链验收。
- `P1`：标准 happy path 无法继续、课程资产缺失、运行时主行动错误、版本变化无法解释。修复后从最近稳定门禁重跑。
- `P2`：降级说明、次要状态、局部布局或非阻断体验错误。记录后可继续，但最终收束前必须解决。
- `P3`：不影响判断和任务完成的文案或轻微视觉问题，可进入后续统一精修。

3. **8.4 跑通主链（已完成）**：按 `ENT -> PRG -> SNP -> REC -> PRA -> DIA -> REM -> CON -> AIT` 执行并保存证据。
4. **8.5A 前端主链组装（已完成）**：用隔离用户执行阅读、精确任务跳转、刷新恢复、正式失败、诊断、补救和独立复验，核对 `UI -> ISO` 的标准 happy path。
5. **8.5B 版本变化（已完成）**：标准课程真实 `cv1 -> cv2` 已证明旧现场可解释迁移、旧任务可解释失效且旧证据不污染新修订。
6. **8.5C 降级（已完成）**：执行 `DEG` 与模型不可用状态；旧课程和阅读型课程不与标准 happy path 混跑。
7. **8.6 缺陷回灌（当前）**：按主归属复盘并修复，课程资产问题回到生成链，协调问题回到 LearningRuntime，展示问题回到前端。
8. **8.7 全量复测**：标准样本从零重跑，旧数据只用于兼容验证；全部 P0/P1 清零后才能宣称第 8 步完成。

出现以下任一情况立即停止 happy path，不得绕过：标准课程没有正式题目、量规或补救资产；课程版本或目标修订为空；正式状态依赖默认用户；验收需要直接修改持久文件；AI 或前端能够越过领域 API 写正式状态。

- 线性代数 smoke test 可以由读取层投影目标修订和内容块修订，但 `current_course_version_id` 与正式练习题目为空。
- 本轮抽查的当前可读课程均未从 `/learning-assets/practice` 返回正式题目，现有课程不能直接承担标准全链路 happy path。
- 当前 ModelScope 凭据返回 401。结构化降级可进入 `AIT-01` 验收，但标准课程真实生成与真实回答质量验收必须在凭据恢复后完成。
- 8.2 已新增共享检查器、单课程 API、全目录 API 和命令行入口。默认 `standard`，只有资产计划显式声明阅读型时 `auto` 才切换到 `reading_only`；缺失资产不会自动降级为兼容课程。

### 14.1 结构化同源演示 A

- 标准课程 `4dcfe257-0955-49bb-ade4-dc6ed915bbfb` 从唯一课程文档编译出大纲、教案、讲义、练习单和幻灯片五类表达，没有保存第二份章节真源。
