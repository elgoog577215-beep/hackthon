# 启智大会双视频正式录制稿

> 2026-07-23 根据李老师反馈重新定稿。本文件是本次大会录制的唯一分镜依据。
> 核心要求：不做画面放大、缩小、追焦或镜头平移，用稳定的完整页面直接证明产品结果。

## 一、两条视频的共同录制原则

两条视频使用同一门《矩阵与线性变换》课程：

- Course ID：`demo-matrix-growth-v2`
- Section ID：`v2-sec-1-2`
- 视频一路由：`/course/demo-matrix-growth-v2/ppt`
- 视频二路由：`/course/demo-matrix-growth-v2/learn/v2-sec-1-2`

画面与后期统一采用以下规则：

1. 固定 `1440 × 810`、16:9、浏览器缩放 `100%`，从开始到结束保持同一画幅。
2. 禁止 Recordly 自动缩放、手工放大、局部追焦、镜头平移和为了强调按钮制造的动效。
3. 只录浏览器内容区，不出现桌面、地址栏、终端、Codex、系统通知和录屏控制框。
4. 原始录屏不带配音、字幕、BGM 和现场声，供李老师现场讲解或后续统一配音。
5. 允许剪掉或加速纯等待过程，但不能跳过“用户确认”，也不能把分析完成剪成自动写入成功。
6. 等待期间鼠标停在右下空白区；操作前自然移动到目标，停顿约 `0.2–0.3s` 后点击。
7. 页面需要滚动时只做一次方向明确的平滑滚动，不来回寻找，不拖动滚动条。
8. 关键结果至少完整停留 `3–5s`，让老师有时间指出“哪里变了、为什么变、哪些没变”。
9. 蓝色表示当前正式课程，黄色表示影响或联动更新，紫色表示个体化生长，绿色只表示独立复验通过。

每次重拍前先停止当前服务，再执行：

```bash
./dev.sh --recordly-demo
```

录制前分别打开两个路由并等待首次加载完成。正式录制从页面稳定后的状态开始，不把加载页录进去。

## 二、视频一：教学材料一键生成及一键更新

### 1. 要讲清楚的唯一主题

**教师只维护一门结构化课程，系统可以一键生成全部教学材料；修改任何一处课程语义，相关材料会精准联动更新。**

它包含两个连续证明：

```text
一份课程结构
→ 一键生成大纲、教案、讲义、PPT、练习册、知识图解

教师在 PPT 中提出一个小修改
→ AI 理解教学语义
→ 分析影响范围
→ 教师确认
→ 相关材料小步更新
→ 不相关内容保持不变
```

视频不是展示“重新生成一套 PPT”，而是展示已有课程材料之间真正有联系，并且能够交互迭代。

### 2. 目标时长

- 建议成片：`2分10秒—2分30秒`
- 硬上限：`3分钟`
- 纯等待累计最多保留 `2秒`；任何连续静止画面不得超过 `3秒`
- 每个镜头都必须承担“操作、过渡或结果证明”之一，不保留无讲解价值的停顿

### 3. 正式分镜

| 时间 | 操作 | 页面必须证明的结果 | 李老师可配合讲解的重点 |
|---|---|---|---|
| 0–3s | PPT 工作台稳定全景，鼠标直接移向「教学材料总览」 | 课程标题、PPT 页面、同源依据和材料入口同时可见 | 传统工具需要分别生成并手工维护 |
| 3–9s | 点击「教学材料总览」 | 出现“一份课程真源·六类教学材料”以及六类材料状态 | 这里维护的不是六份孤立文件，而是一份课程结构 |
| 9–24s | 连续点击大纲、教案、讲义、演示文稿、练习册、知识图解，每项约 2–3s | 六类材料均有真实内容与“同源已连接” | 一键生成之后，所有材料仍然知道自己来自哪里 |
| 24–31s | 返回 PPT，选中目标页并进入核心信息修改 | 修改入口、原学习目标和课程来源同时可见 | 下面演示已有材料的交互迭代 |
| 31–45s | 把旧目标替换为固定新目标，立即点击「分析影响」 | `掌握矩阵乘法的计算规则` 变为固定新目标 | 只改一个教学目标，不是整套重做 |
| 45–58s | 查看语义变化与范围 | 明确出现 `计算技能 → 概念理解`、预计联动数量和保持不变数量 | 系统理解的是教学含义，不是简单文字替换 |
| 58–67s | 点击「改变课程含义并联动」 | 出现修改前后、理由、影响范围和待确认状态 | AI 先规划，教师确认后才执行 |
| 67–75s | 点击「确认并同步相关内容」；真实等待只保留 1–2s | 出现真实同步回执：实际更新、仅校验、确认无需处理 | 确认后才形成课程修订和材料重编译 |
| 75–96s | 按大纲、教案、讲义、PPT、练习快速切换，每类约 3–4s | 每类都出现具体前后差异或来源校验结果 | 大纲改得最少，讲义更细，PPT 与练习按各自形式更新 |
| 96–104s | 点击「查看更新后的课件」并回到目标页 | PPT 页面真实显示新目标，改动位置高亮 | 先证明课件本身真的变了 |
| 104–114s | 进入同一课程的学习正文 | 对应正文块自动定位并显示黄色 `PPT 同源更新` 高亮与前后差异 | 再证明课程正文不是旧版本 |
| 114–123s | 返回材料总览，在讲义中选择一个例子并点击「编辑并联动」 | 出现原讲义小段与局部编辑框 | 同源不只支持“大版本更新”，也支持日常小改 |
| 123–134s | 输入一行更清楚的例子并点击「分析影响」 | 明确显示局部只影响 `6` 个同源单元，约 `120` 个单元保持不变 | 小改只沿真实依赖传播，不重新生成每一页 |
| 134–143s | 确认同步并查看局部回执 | 六类材料各命中对应单元，其余单元复用 | 这就是可持续的交互迭代 |
| 143–148s | 回到更新后的讲义或课程正文，停留 3–5s | 新增行高亮、AI 更新标记和正式内容同屏 | 结果导向：观众最后看到真实更新后的材料 |

### 4. 固定操作文本

修改前：

> 掌握矩阵乘法的计算规则

修改后：

> 理解矩阵乘法为什么表示线性变换的复合，并能解释运算顺序

### 5. 视频一验收门禁

- 开头必须看见六类材料，而不只是 PPT。
- 必须出现 `计算技能 → 概念理解`，证明系统理解教学语义。
- 必须先出现候选和影响范围，再由教师确认写入。
- 同步完成后，至少实际打开大纲、教案、讲义、PPT、练习五类结果。
- 每类都要看到具体差异或来源校验，不能只报一个“已完成”数字。
- 必须保留“保持不变/确认无需处理”，证明不是整门课程从零重做。
- 大更新后必须返回真实 PPT，再进入同一课程正文查看黄色高亮；回执不能作为结尾。
- 必须再演示一次讲义局部小改，影响范围应明显小于大更新，并展示大量单元保持不变。
- 不展示「清风课堂」「学术蓝灰」等与主题无关的主题按钮。

## 三、视频二：学生个体化学习与课程生长

### 1. 要讲清楚的唯一主题

**学生不是在固定课程里被动学习；学生暴露出的真实困难，会在确认后让自己的课程长出解释、动画、练习和后续承接，并由新的正式题验证调整是否有效。**

完整证明链：

```text
学生说清楚会什么、卡在哪里、希望怎样学
→ AI 识别四类学习语义
→ 生成当前小节及相关后续的课程生长方案
→ 学生选择范围并确认
→ 正式课程长出新内容
→ 动画帮助理解
→ 针对性正式题独立复验
```

### 2. 目标时长

- 建议成片：`1分30秒—2分钟`
- 硬上限：`3分钟`
- 不为了追求 30 秒而牺牲四类语义、影响范围和独立复验

### 3. 固定演示输入

> 矩阵乘法计算我会，但我一直不理解为什么复合变换要先右后左。请在本节和后面相关内容中，先用几何动画解释，再让我进行计算。

### 4. 正式分镜

| 时间 | 操作 | 页面必须证明的结果 | 李老师可配合讲解的重点 |
|---|---|---|---|
| 0–7s | 学习页稳定全景，不操作 | 左侧完整课程目录、中央当前课程、右下 AI 老师入口同时可见 | 同一门课面对不同学生，不应永远只有一个版本 |
| 7–13s | 点击「智能助教」 | AI 老师面板打开，左侧目录仍然可见 | 学习行为发生在真实课程现场 |
| 13–25s | 在输入框粘贴固定文案并发送 | 原句完整可见后再发送 | 一句话同时说明已会内容、困难、教学要求和影响范围 |
| 25–42s | 不操作，查看强证据卡和方案 | 出现“已识别强学习证据”；显示已会、困难、教学要求、影响范围；方案包含 6 项变化 | AI 不只是回答问题，而是在理解学生的知识状态 |
| 42–50s | 点击「本小节及后续」 | 范围选择明确；左侧相关节点出现建议或影响标记 | 学生可以决定只改当前，还是连同相关后续一起调整 |
| 50–57s | 点击「整体确认并更新课程」 | 用户确认动作清楚 | 课程不会在学生不知情时被自动改写 |
| 57–70s | 鼠标移开，等待目录闪烁和正文自动定位 | 目录转为蓝色“已应用”；正文出现紫色 AI 个体化补充、动画和练习 | 课程不是弹出一段回答，而是真的长出了新结构 |
| 70–82s | 点击「播放动画」，等待至 `A(Bv)` | 动画依次出现 `v → Bv → A(Bv)` | 先把抽象顺序变成可观察的状态变化 |
| 82–88s | 选择动画判断「先 B，再 A」 | 出现正确反馈 | 学生先完成过程内理解检查 |
| 88–103s | 点击「开始练习」，选择 A，点击「提交作答」 | 正式题得分 100，显示达到本题标准 | 追加练习不是堆题，而是独立验证刚才的理解 |
| 103–112s | 点击「学习证据轨迹」 | 原始自述、AI 问答、动画反馈和正式练习按时间可追溯 | 个性化不是模型猜测，每次变化都能回到证据 |
| 112–124s | 点击「个人学习地图」 | 全课节点区分按原路径、已补充、已升级、已折叠或已重组；本次相关节点显示已补充 | 每个人最终留下的是一份自己的学习路径和成长教材 |
| 124–130s | 保持地图和绿色复验结果同屏 | 绿色“本轮独立复验通过”、受影响路径与未改变路径同时可见 | 一次通过只形成初步支持，系统还会继续观察后续迁移 |

### 5. 视频二验收门禁

- 左侧目录在关键阶段保持可见，证明变化发生在完整课程结构中。
- 强证据卡必须完整显示四类语义：已会、困难、教学要求、影响范围。
- 必须展示“只应用本小节”和“本小节及后续”的范围选择。
- 用户确认前正式课程不能改变；确认后目录和正文必须连续出现明显反应。
- 必须实际播放结构化动画，并完成动画内判断。
- 必须完成新增正式练习，不能以“等待复验”结束视频。
- 绿色只能出现在正式练习通过之后。
- 结尾必须打开证据轨迹与个人学习地图，证明“个性化”形成了可追溯路径，而不是一次 AI 回答。
- 结尾保留“仍需后续不同任务持续确认”，不夸大一次作答的结论。

## 四、正式录制顺序

为避免第一次视频的课程更新污染第二条视频，推荐按以下顺序录制：

1. 停止服务，执行 `./dev.sh --recordly-demo`。
2. 先录视频一；录完后停止服务。
3. 再执行一次 `./dev.sh --recordly-demo`，恢复干净课程和学生状态。
4. 录视频二；若重拍，从第 3 步重新开始。
5. 后期只做开头结尾裁切、等待压缩和必要的两段拼接，不做任何画面缩放。
