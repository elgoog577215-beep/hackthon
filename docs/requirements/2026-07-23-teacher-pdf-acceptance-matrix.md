# 李老师原始需求 PDF 全链路验收矩阵

> 来源：`artifacts/需求对齐/“针对性地教、个性化地学”功能与演示需求理解确认稿.pdf`
>
> 本矩阵把老师的口述和 PDF 转成“产品能力、真实链路、视频证据”三重验收。只有代码、
> 接口、正式数据变化和浏览器结果同时成立，才能标记为完成；单独存在页面或演示文案
> 不算完成。

## 一、总体判断

老师的要求不是两个孤立功能，而是同一套课程智能的教师侧与学生侧：

```text
教师侧：针对性地教
课程结构 → 六类材料 → 任意材料修改 → 语义影响分析 → 教师确认
→ 相关材料精准更新 → 返回真实 PPT 与课程正文证明结果

学生侧：个性化地学
原始学习证据 → AI 形成可解释判断 → 生成增、改、减、重组方案
→ 学生选择范围并确认 → 当前与后续课程真实变化 → 独立复验
→ 留下个人学习地图与成长教材
```

## 二、教师侧“教学材料一键生成及一键更新”

| 老师要求 | 正式产品能力 | 当前实现与证据 | 视频必须出现 | 状态 |
|---|---|---|---|---|
| 一份课程生成全部材料 | 同一 `CourseDocument` 编译大纲、教案、讲义、PPT、练习、图解 | 教学材料总览读取同一课程的六类正式表示 | 六类材料逐类打开，不能只报数量 | 已完成 |
| 大纲向下生成讲义与 PPT | 大纲/教案学习目标作为课程语义入口，修改后沿真实依赖传播 | 大纲“编辑并联动”复用统一影响分析与同步回执 | 大纲修改后展示讲义、PPT、练习的真实差异 | 已完成 |
| PPT 反向更新讲义和大纲 | PPT 语义编辑先解析教学维度，再形成课程候选 | 固定目标可识别 `计算技能 → 概念理解` 并产生课程修订 | PPT 修改、语义变化、范围、确认、回执 | 已完成 |
| 不是简单替换文字 | 影响分析以课程块、目标、知识和表示单元为边界 | 回执区分实际更新、仅校验与确认无需处理 | 同屏出现“更新了什么”和“哪些保持不变” | 已完成 |
| 不是每次整套重新生成 | 讲义小段修改只命中对应课程块和六类派生单元 | 局部讲义样本实测 `6` 个受影响、约 `120` 个不变 | 大更新后再做一次一行级小改 | 已完成 |
| 正向与逆向都要联动 | PPT、大纲、教案、讲义入口共用课程语义协议 | 各入口均先影响分析、再确认、再同步 | 至少演示一次反向和一次局部正向 | 已完成 |
| 修改后结果必须看得见 | 同步后可返回真实课件并进入课程正文 | PPT 目标真实更新；正文自动定位并显示黄色同源高亮 | 回执后返回 PPT，再进入正文看高亮 | 已完成 |
| 教师拥有最终决定权 | AI 只生成候选；确认前课程不变 | 支持确认、拒绝与影响边界预览 | 不能把分析完成剪成自动写入 | 已完成 |
| 面向教师“针对性地教” | 材料可以围绕学生反馈做小步迭代 | 同一语义协议支持日常小改，不必从零备课 | 结尾强调下一次课可继续交互迭代 | 已完成 |
| 教案与 PPT 最终形成教材 | 当前以讲义/课程正文承载图文材料，教材是后续产品形态 | 尚未新建独立教材真源，避免概念与数据重复 | 本次可由老师口头展望，不作为当前 Demo 功能宣称 | 后续范围 |

## 三、学生侧“个性化地学”

| 老师要求 | 正式产品能力 | 当前实现与证据 | 视频必须出现 | 状态 |
|---|---|---|---|---|
| 感知学生学习过程 | AI 对话、反馈、笔记、错题、复习任务和正式练习进入 `EvidenceItem` | 三类来源、细分证据类型、时间与课程锚点均保留 | 打开“学习证据轨迹” | 已完成 |
| 一句话理解学生状态 | 强自述合同识别已会、困难、讲法和范围 | 固定输入可立即形成强证据方案 | 四类语义必须同时可见 | 已完成 |
| 不用固定次数机械触发 | 综合直接性、独立来源、正式证据、反证和范围 | 单条明确强证据可局部触发；跨章节要求更强证据 | 显示证据强度和范围理由 | 已完成 |
| 当前小节主动长出内容 | `CourseEvolutionPlan` 支持解释、动画、检查和正式练习 | 确认后写入新的正式课程修订 | 正文出现紫色新增块 | 已完成 |
| 提前修改后续课程 | 方案允许 `current_and_next`，目录显示相关后续节点 | 固定输入生成当前位置与相关后续六项变化 | 点击“本小节及后续”，目录出现标记 | 已完成 |
| 可以增加与改写 | `INSERT_COURSE_BLOCK / REPLACE_COURSE_BLOCK` | 支持新增块与保留稳定块身份的升级 | 方案预览显示新增与升级 | 已完成 |
| 可以删减或跳过已会内容 | `FOLD_COURSE_BLOCK` 在确认后退休默认支架，历史仍保留 | 多次独立正式通过的挑战方案才允许提出折叠 | 地图可显示“已折叠”；无需塞进本次主链操作 | 已完成 |
| 可以重组学习顺序 | `REORDER_COURSE_BLOCK` 只允许同一小节内移动稳定块 | 记录原位置，确认后原子移动，撤销恢复原顺序 | 地图可显示“已重组” | 已完成 |
| 能力提高后增加挑战 | 正式重复通过生成挑战建议，保留旧难度掌握 | 挑战方案增加理论/迁移任务，并可减少脚手架 | 可作为备用镜头，不打断本次“困难→生长”主线 | 已完成 |
| 用户决定范围和内容 | 支持当前、当前及后续、多节点逐项选择 | 未选节点不写入；确认、拒绝、撤销均有正式回执 | 范围切换与确认动作清楚可见 | 已完成 |
| 变化理由可解释 | 每个方案与操作保留证据、理由、范围和预期效果 | 支持查看依据、回到证据位置 | 证据轨迹与方案理由前后呼应 | 已完成 |
| 新内容需要复验 | 动画内判断只作过程证据；正式题独立验证效果 | 固定正式题确定性评分，100 分后才显示绿色 | 必须做完正式题再出现绿色 | 已完成 |
| 每个人留下不同路径 | 个人学习地图投影保留、补充、升级、折叠和重组状态 | 地图来自当前课程、已应用方案和当前学习进度，不复制正文 | 视频结尾打开地图 | 已完成 |
| 只影响当前学生和当前课程 | 所有证据和方案按 `user_id + course_id` 保存 | 不跨课程传播，不改共享知识或历史原文 | 地图底部显示隔离说明 | 已完成 |

## 四、录制与展示验收

| 验收项 | 通过标准 |
|---|---|
| 节奏 | 连续静止画面不超过 3 秒；纯等待累计不超过 2 秒；等待用跳剪或加速 |
| 结果导向 | 每次确认后必须回到真实材料或课程正文，不能把统计回执当最终结果 |
| 教师视频 | 同时证明一键生成、大语义更新、局部小改、真实 PPT 与正文变化 |
| 学生视频 | 同时证明证据、判断、范围、确认、课程生长、后续节点、正式复验与个人地图 |
| 画面 | 固定 16:9、100% 缩放，不做放大、缩小、追焦和无意义镜头移动 |
| 演示数据 | 录制前重置专用课程与专用学习者；页面首次加载在录制前完成 |
| 真实性 | 可以预置高质量起点，但确认、课程修订、材料同步、课程生长和复验必须走正式链路 |

## 五、验收结论与录制前门禁

截至 2026-07-23，PDF 中属于本期双 Demo 的正式产品能力已经完成开发与纵向验收：

- 教师侧已从六类材料总览走通大纲向下联动、PPT 反向联动、讲义局部小改、逐材料差异回执、返回真实 PPT 和进入课程正文查看同源高亮。
- 学生侧已从强学习自述走通语义识别、六节点逐项预览、确认写入、当前正文与三个后续小节变化、动画过程证据、100 分正式独立复验、证据轨迹和四节点个人学习地图。
- “全课程”入口遇到强学习证据时，审阅工作台会绑定真实生长方案，不再停留在空扫描态。

开发验收已经完成，下一阶段不再补产品主链，而是做录制准备：执行专用数据重置、提前加载
两个固定路由、按分镜踩点计时，并确认正式录制过程中连续静止不超过 3 秒。独立“教材”
产品仍按老师原话属于未来展望，不纳入本次大会 Demo 的完成口径。
