# 2026-07-19 答辩演示视频需求整理

来源：产品负责人 2026-07-19 口述 + 两条视频最终优化版脚本。
两条视频共用同一门**新生成**的《矩阵与线性变换》课程（demo 无法在旧课程上录制）。

共同母题：**课程开始理解变化——它理解教师为什么改，也理解学生为什么卡。**

## 统一视觉颜色契约（两条视频强制）

| 颜色 | 含义 |
|---|---|
| 蓝色 | 当前正式课程 |
| 黄色 | 受到影响的位置 |
| 紫色 | AI 待确认方案 |
| 绿色 | **仅**经过后续学习复验（独立正式证据）的变化 |

红线：“已应用”**不得**画成绿色；绿色只留给真正复验通过的状态（当前实现中
`effect_evaluation.status === 'effective'` 才是绿色，“已应用未复验”为蓝色——符合要求）。

## 视频一：结构化同源（55–65 秒）

标题：理解教师的修改——一处改变，全课同源。

演示主链路（必须全部真实可操作）：

```
教师修改 → 理解教学语义 → 分析依赖范围 → 教师确认 → 相关内容精准联动
```

分镜要求：

1. PPT、教案、课程正文、题目并列出现（字幕：改了一页 PPT，其他内容怎么办？）
2. PPT 工作台中把学习目标从「掌握矩阵乘法的计算规则」改为
   「理解矩阵乘法为什么表示线性变换的复合，并能解释运算顺序」——只展示这一个动作。
3. 点击「分析影响」→ 系统必须识别出**计算技能 → 概念理解**的教学维度跃迁
   （证明不是文字 diff）。
4. 影响范围图：教案重点 / 正文解释 / 动态图解 / 例题 / 理解检查**标黄**；
   无关章节保持**蓝色**。
5. 展示修改前后差异；教师点击「同步至相关内容」；明确显示修改理由、影响范围、
   不受影响内容。
6. 相关资源依次更新，出现「**5 项已同步，12 项无需修改**」一类统计结果。
7. 落版：结构化同源：一处改变，全课联动。

核心证明点：**该动的动，不该动的不动**（不是全部自动重生成）。

## 视频二：个体化生长（60–70 秒）

标题：理解学生的学习——一句话，课程开始生长。

演示主链路：

```
学生表达真实困难 → AI 理解四类学习语义 → 生成当前与后续课程方案
→ 学生确认 → 正式课程长出新内容 → 动画呈现明显学习反应
```

固定演示输入（学生在「1.2 矩阵」小节的 AI 老师面板输入）：

> 矩阵乘法计算我会，但我一直不理解为什么复合变换要先右后左。请在本节和
> 后面相关内容中，先用几何动画解释，再让我进行计算。

系统必须**明显展示**四个语义维度：

- 已会内容：矩阵乘法计算
- 持续困难：不理解复合变换顺序
- 教学要求：几何动画解释、再进行计算
- 影响范围：本节及相关后续内容

分镜要求：

1. 左侧课程目录全程保留可见。
2. 发送后**立即**出现「已识别强学习证据」+ 四维度展示。
3. 课程生长方案展开：当前位置解释、分步动画、针对性练习、后续过渡、理解检查。
4. 左侧当前小节及相关后续节点变**紫色**；提供「本小节及后续」与
   「只应用本小节」两种范围选择。
5. 点击「整体确认并更新课程」→ 目录显示「已应用」（蓝），课程生成新版本。
6. 新增动画可播放：依次展示 `v → Bv → A(Bv)`，解释复合变换为什么先右后左。
7. 结尾状态：课程新版本已应用、可以撤销、等待独立复验。

### 演示触发条件（产品决策，已确认）

个体化生长的触发条件**允许自由设置**（演示模式放宽阈值），否则录屏无法稳定
触发。实现要求：通过配置（环境变量或课程级设置）放宽/直通强证据判定，而不是
改死生产逻辑。
