# 2026-07-19 答辩演示视频需求整理

来源：产品负责人 2026-07-19 口述 + 两条视频最终优化版脚本。
两条视频围绕同一门《矩阵与线性变换》的教学主题展开。视频一使用动态创建并发布的
正式课程；为避免生成质量、模型配额和历史学习状态影响视频二，本次录屏决策允许
视频二使用手工定稿且可重复重置的专用课程快照，不把两者宣称为同一个 `courseId`。

共同母题：**课程开始理解变化——它理解教师为什么改，也理解学生为什么卡。**

## 统一视觉颜色契约（两条视频强制）

| 颜色 | 含义 |
|---|---|
| 蓝色 | 当前正式课程 |
| 黄色 | 受到影响的位置 |
| 紫色 | AI 待确认方案 |
| 绿色 | **仅**经过后续学习复验（独立正式证据）的变化 |

红线：“已应用”**不得**画成绿色；绿色只留给真正复验通过的状态（当前实现中
`effect_evaluation.status === 'effective'` 才是绿色，“已应用未复验”为蓝色——符合要求）。

## 视频一新课与纵向验收契约

视频一从创建到 PPT、学习工作区和导出始终使用同一个 `courseId`。标准样本为
4 章 12 节的《矩阵与线性变换》；目录确认前必须把“矩阵乘法”小节目标校对为
「掌握矩阵乘法的计算规则」，保证后续语义编辑有确定基线。视频二直接进入同主题的
录屏专用快照，不经过这条动态生成链：

```text
新建课程并确认需求
→ 检查 4 章 12 节目录并确认
→ 冻结全课知识职责、分批教案、知识库/关系图、逐节正文
→ 确认发布
→ 从同一课程进入 PPT 工作台
→ 生成同源课件
→ 学习目标语义编辑
→ 影响分析与教师确认
→ 精准同步
→ 导出 PPTX / 失败时保留上一可用版本
```

录制时必须留存以下可核验事实：

| 检查点 | 通过条件 |
|---|---|
| 新课身份 | 创建后直接进入学习现场；地址栏 `courseId` 在后续 PPT 路由中保持一致 |
| 目录门 | 明确显示 4 章 12 节；只有目录和发布需要用户确认 |
| 教案与知识 | 任务状态展示骨架、批次、小节和汇编真实进度；知识库与关系图来自同一正式教案 |
| 恢复 | 失败后继续原任务和原课程，复用已保存检查点，不新建重复课程 |
| 正式发布 | 点击「确认并发布」后同一 URL 原地成为正式学习页 |
| PPT 同源 | 课件页显示课程来源、知识/能力或题目绑定，并通过质量检查 |
| 语义回流 | 修改先形成影响预览和候选；教师确认前课程真源不变 |
| 精准同步 | 回执区分实际更新、仅校验和无需处理；无来源关系的内容保持不变 |
| PPTX 导出 | 当前正式课件可下载，在原生 PowerPoint 中页数一致、中文/公式可读、对象可编辑、备注存在 |
| 失败兜底 | 已有正式课件时，同步失败明确显示上一可用版本；临时问题页不得覆盖或导出 |

外部 PPTX 回导不在本次 Demo 范围内。原生 PowerPoint 可编辑只表示交付文件可继续使用，
不能宣传为可直接回写课程；评估结论见
[外部 PPTX 再导入评估](../architecture/external-pptx-reimport-evaluation.md)。

## 视频一：结构化同源（55–65 秒）

标题：理解教师的修改——一处改变，全课同源。

演示主链路（必须全部真实可操作）：

```
教师修改 → 理解教学语义 → 分析依赖范围 → 教师确认 → 相关内容精准联动
```

分镜要求：

1. 用同一 `courseId` 的新课发布页和 PPT 路由做一个快速转场；PPT、教案、课程正文、
   题目并列出现（字幕：改了一页 PPT，其他内容怎么办？）。
2. PPT 工作台中把学习目标从「掌握矩阵乘法的计算规则」改为
   「理解矩阵乘法为什么表示线性变换的复合，并能解释运算顺序」——只展示这一个动作。
3. 点击「分析影响」→ 系统必须识别出**计算技能 → 概念理解**的教学维度跃迁
   （证明不是文字 diff）。
4. 影响范围图：教案重点 / 正文解释 / 动态图解 / 例题 / 理解检查**标黄**；
   无关章节保持**蓝色**。
5. 展示修改前后差异；点击「展开完整影响图」，明确显示修改理由、影响范围和不受影响
   内容。
6. 点击「改变课程含义并联动」，待候选出现后再点击「确认并同步相关内容」；不能把
   分析完成直接剪成自动写入成功。
7. 相关资源依次更新，回执显示「实际更新 / 仅校验 / 确认无需处理」三类真实统计，
   数量以本次新课程结果为准，不写死为 5/12。
8. 点击「导出 PPTX」，短镜头展示原生 PowerPoint 中可编辑文本框和讲者备注。
9. 落版：结构化同源：一处改变，全课联动。

核心证明点：**该动的动，不该动的不动**（不是全部自动重生成）。

如需拍摄故障兜底，必须先有一版质量通过的正式课件，再模拟同步请求失败；镜头中同时
保留原页面并出现「本次生成失败，当前展示上一可用版本」。恢复网络后重新同步再导出，
不要把离线下载失败当成课件导出失败。完整操作见 `docs/demo-path.md`。

## 视频二：个体化生长（60–70 秒）

标题：理解学生的学习——一句话，课程开始生长。

演示主链路：

```
学生表达真实困难 → AI 理解四类学习语义 → 生成当前与后续课程方案
→ 学生确认 → 正式课程长出新内容 → 动画呈现明显学习反应
```

固定演示输入（学生在「1.2 矩阵」小节的 AI 老师面板输入）：

> 矩阵乘法计算我会，但我一直不理解为什么复合变换要先右后左。请在本节和
> 后面相关内容中，先用几何动画解释，再让我进行计算。

系统必须**明显展示**四个语义维度：

- 已会内容：矩阵乘法计算
- 持续困难：不理解复合变换顺序
- 教学要求：几何动画解释、再进行计算
- 影响范围：本节及相关后续内容

分镜要求：

1. 左侧课程目录全程保留可见。
2. 发送后**立即**出现「已识别强学习证据」+ 四维度展示。
3. 课程生长方案展开：当前位置解释、分步动画、针对性练习、后续过渡、理解检查。
4. 左侧当前小节及相关后续节点变**紫色**；提供「本小节及后续」与
   「只应用本小节」两种范围选择。
5. 点击「整体确认并更新课程」→ 目录显示「已应用」（蓝），课程生成新版本。
6. 新增动画可播放：依次展示 `v → Bv → A(Bv)`，解释复合变换为什么先右后左。
7. 结尾状态：课程新版本已应用、可以撤销、等待独立复验。

### 演示触发条件（产品决策，已确认）

个体化生长的触发条件**允许自由设置**（演示模式放宽阈值），否则录屏无法稳定
触发。实现要求：通过配置（环境变量或课程级设置）放宽/直通强证据判定，而不是
改死生产逻辑。
