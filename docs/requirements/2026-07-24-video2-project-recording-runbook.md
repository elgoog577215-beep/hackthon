# 视频二录制执行单：项目任务生成个人学习路径

> 本执行单替代《灵知大会双视频最终分镜（教师确认版）》中旧的“Prompt 升级为论文助手”视频二方案。视频一不变。

## 核心证明

学生面对同一个项目任务时，系统不是统一加难题，而是根据学生自述的已有经验和当前短板，生成一份暂定个人路径：已会内容进入项目验证，薄弱内容重点补充，交付物拆成项目里程碑。自述不等同于掌握，路径仍需学生确认。

固定项目：设计一个可验证的 AI 论文助手，研究“AI 辅助学习是否提高大学生学习效果”。

固定输入：

- 交付物：一套论文检索、证据比较和引用核验流程，以及一份带来源的研究报告。
- 已有经验：会写提示词，能限定资料范围、规定输出格式并要求标注原文依据。
- 当前短板：不会设计多步骤检索，不会筛选可靠论文，也不确定怎样核验结论和引用。

## 91 秒画面节奏

1. `0–11 秒`：课程库进入新建课程，选择“项目实战”。
2. `11–45 秒`：依次输入项目目标、交付物、已有经验和当前短板。每次输入前鼠标停顿，输入后保留约一秒。
3. `45–57 秒`：展示“暂定学习起点”说明，点击“确认需求，生成目录”。
4. `57–66 秒`：展示交付物、已有经验和重点补充三项起点信息。
5. `66–73 秒`：打开“提示词限定能力实战验证”，证明已会内容不重复讲授，也不直接认定掌握。
6. `73–80 秒`：展示“研究问题拆解与检索策略设计”，证明短板被设为重点补充。
7. `80–89 秒`：展示“引用来源可验证性核查”和“流程封装与最终交付”。
8. `89–91 秒`：鼠标停在“确认目录并继续”，说明学生确认后才继续生成正式课程；落版“同一个项目，每个人都能长出自己的学习路径”。

## 录制前检查

- 只捕获 Microsoft Edge 窗口，不关闭微信或其他应用。
- Edge 使用 `http://127.0.0.1:5175`，后端使用 `http://127.0.0.1:8001`，调试端口为 `9223`。
- Recordly 已选中 Edge 窗口，开始/停止按钮仍位于主屏幕逻辑坐标 `(950, 949)`。
- 浏览器缩放 `100%`，完整窗口保持 16:9；关闭系统通知。
- 鼠标由真实系统事件控制；画面使用加大光圈、移动轨迹、点击涟漪和操作箭头。
- 正式录制前先运行干跑，四类路径证据必须全部出现。

## 执行命令

干跑检查，不启动录制：

```bash
node scripts/record_video2_project_path.mjs --dry-run
```

正式录制：

```bash
node scripts/record_video2_project_path.mjs
```

正式脚本不会点击“确认目录并继续”，因此同一份样板课程可以重复录制，不会被下一阶段生成污染。
