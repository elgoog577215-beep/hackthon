# 学科教学模板 V2 设计依据

## 结论

学科模板不应是一组每节重复出现的栏目。更有效的结构是：

```text
一级学科模式
→ 必要学科分型
→ 当前小节课型
→ 课型规定的学习行为、课程块和成果证据
→ 编排偏好与难度在其上叠加
```

一级模式回答“这门课主要通过什么行为学习”，分型只处理同一大类中确有方法差异的
领域，课型则回答“当前小节具体怎样教”。三层都由本地规则根据已确认主题、小节目标和
课程阶段确定，不新增模型调用。

## 跨学科设计依据

- 计算教育不能只覆盖知识点。ACM/IEEE 的计算课程框架把能力放在任务情境中理解，
  同时考虑知识、技能和专业倾向。因此编程课需要运行、设计、调试、测试、评审和取舍，
  不能只做“代码示例 + 解释”。
  来源：[Computing Curricula 2020](https://www.acm.org/binaries/content/assets/education/curricula-recommendations/cc2020.pdf)、
  [CS2023 能力框架](https://csed.acm.org/wp-content/uploads/2025/11/CS2023-Report.htm)。
- 数学学习需要目标明确的问题、表征之间的连接、推理表达、例题策略、变式和有支持的
  认知挑战，直觉不能替代正式定义。
  来源：[NCTM Principles to Actions](https://www.nctm.org/store/Products/Principles-to-Actions--Ensuring-Mathematical-Success-for-All/)、
  [NCTM Productive Struggle](https://www.nctm.org/Conferences-and-Professional-Development/Professional-Services/Productive-Struggle/)。
- 科学教育的重点是让学习者使用实践理解现象：提出问题、建立和使用模型、调查、分析
  数据、从证据论证，并用核心概念和跨领域概念解释或设计。
  来源：[NGSS Science and Engineering Practices](https://www.nextgenscience.org/glossary/science-and-engineering-practices-sep)、
  [NGSS 三维学习说明](https://www.nextgenscience.org/glossary?first_letter=t&name=)。
- 生命科学需要把核心概念与科学实践结合：跨尺度连接结构与功能、信息、物质能量、
  系统和演化，同时使用定量推理、模型、实验与跨学科沟通。医学基础课程则必须把基础
  机制转化为可观察能力，并保持案例推理的安全边界。
  来源：[NSF Vision and Change](https://www.nsf.gov/news/vision-change-undergraduate-biology-initiative)、
  [AAMC Competency-Based Medical Education](https://www.aamc.org/about-us/mission-areas/medical-education/cbme)。
- 社会研究的成熟路径是“提出问题 → 使用学科概念和工具 → 评价来源并使用证据 →
  表达结论或采取有依据的行动”。历史、文本、哲学和社会科学仍需不同的材料与论证课型，
  不能合并成一个固定的人文栏目。
  来源：[NCSS C3 Framework](https://www.socialstudies.org/sites/default/files/c3/c3-framework-for-social-studies-rev0617.pdf)。
- 语言学习应围绕现实行动和“能够做什么”组织，覆盖理解、产出、互动和调解；形式教学
  服务意义与使用，输出需要反馈、修正和再次表现。
  来源：[Council of Europe CEFR](https://www.coe.int/en/web/portfolio/the-common-european-framework-of-reference-%20%20%20%20%20%20%20%20%20for-languages-learning-teaching-assessment-cefr-)、
  [CEFR 行动导向指南](https://rm.coe.int/a-guide-to-action-oriented-plurilingual-and-intercultural-education-en/1680b52354)。
- 商业和职业课程应以成果和经验学习为中心，要求学习者诊断问题、使用工具、在不完整
  信息下决策、完成工作交付、获得反馈，并考虑伦理与利益相关者影响。
  来源：[AACSB Global Standards](https://www.aacsb.edu/educators/global-standards)。
- 通用课程作为兜底，也不能退化成百科文章。它至少要区分概念建构、方法演练、比较
  判断、案例应用和综合迁移，并允许不同的理解与表达方式。
  来源：[CAST UDL Guidelines](https://udlguidelines.cast.org/action-expression/)。

## V2 课型结构

| 一级模式 | 内部必要分型 | 五类核心课型 |
|---|---|---|
| 通用课程 | 概念理解、方法应用 | 概念建构、方法工作坊、比较判断、案例应用、综合迁移 |
| 数学与形式科学 | 理论证明、计算建模 | 直觉与表征、解题策略、证明论证、错误诊断、建模探究 |
| 编程与工程技术 | 编程基础、软件系统、数据与 AI | 最小运行、引导实现、调试实验、测试重构、项目架构 |
| 自然科学 | 物理模型、实验探究、环境应用 | 现象探究、模型解释、实验调查、证据论证、设计应用 |
| 生命科学与医学基础 | 生命系统、医学基础 | 结构功能、机制系统、生物证据、比较变异、案例推理 |
| 人文社科 | 历史、文本文化、哲学论证、社会科学 | 问题语境、材料解释、因果变化、观点论证、综合表达 |
| 语言学习 | 日常交际、学术职业沟通 | 可理解输入、形式准确、互动任务、调解转述、表现修正 |
| 商业与职业技能 | 战略运营、市场沟通、分析财务 | 场景诊断、案例决策、工具工作坊、角色模拟、交付复盘 |

每个课型必须同时声明：

- 当前小节要造成的学习变化。
- 必须出现的学科课程块。
- 可观察的成果证据。
- 最容易退化成模板内容的质量底线。
- 更适合出现的课程阶段。

## 稳定性边界

- 课型只决定本节课程块与讲法，不改目录、知识身份、资料范围和难度合同。
- 学科分型与课型选择都由确定性代码完成，不新增分类模型或生成步骤。
- 编排偏好只能在学科课型之上补充案例、项目、探究等节奏，不能删除课型必需块。
- 旧课程不迁移；只有新生成或明确重新生成的课程使用 V2。
- 是否提升真实内容质量仍需同主题、同模型、同资料的跨学科成课对照验证。
