# 添加统一学习事件账本

## Why

当前后端已经有课程生成、AI 助手、标注笔记、导师记忆和测验/错题能力，但这些学习行为分别散落在课程 JSON、`annotations.json`、`tutor_memory.json` 和即时接口返回中。系统还无法稳定回答“用户刚刚产生了什么学习证据”，因此后续学习者状态模型和教学决策层缺少统一输入。

## What Changes

- 新增轻量 `LearningEvent` 账本，继续使用当前文件存储，不引入数据库或新依赖。
- 统一记录事件类型、用户、课程、节点、概念线索、来源、证据、结果和元数据。
- 首批接入 3 个高价值入口：AI 问答、标注/笔记保存、导师学习结果记录。
- 保持旧接口返回兼容，不改前端协议。

## Out of Scope

- 不建立完整学习者状态模型。
- 不实现教学决策层。
- 不改前端 UI。
- 不一次性接入所有节点重写、课程生成、复习建议入口。
