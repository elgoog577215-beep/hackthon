## REMOVED Requirements

### Requirement: 教学标准库必须区分能力易错与提升

系统 MUST 将能力点、易错点和提升点保存为不同正式对象。能力点 MUST 关联一个主知识节点；易错点 MUST 归属于能力点；提升点 MUST 归属于能力点或知识节点。

#### Scenario: 查询高斯消元教学标准

- **WHEN** 系统读取高斯消元相关标准
- **THEN** 响应 MUST 区分可观察能力、具体易错行为和进阶训练方向
- **AND** 易错点 MUST 包含错误表现、认知原因和辨析或修复方法

### Requirement: 标准库只服务 AI 不限制 AI

AI MUST 先依据当前课程、任务和证据形成判断，再使用标准库统一命名、路径和颗粒度。系统 MUST 允许命中、部分命中和未命中，不得为了填入正式 ID 强迫错误映射。

#### Scenario: 当前错误未被标准库覆盖

- **WHEN** AI 有充分证据发现真实问题但正式标准库没有对应条目
- **THEN** AI MUST 保留该问题并标记为部分命中或未命中
- **AND** 系统 MUST NOT 伪造正式标准 ID

### Requirement: 学生端知识视图必须显示学科语义与课程覆盖

学生端 MUST 展示学科语义路径，并叠加当前课程正文、题目、掌握标准和误区绑定。学生端 MUST NOT 用课程、章节和小节重新构造知识树。

#### Scenario: 学生从线性代数课程打开知识库

- **WHEN** 学生打开当前课程知识入口
- **THEN** 根节点 MUST 是学科或领域语义节点
- **AND** 学生 MUST 能看到当前课程覆盖标记并跳回相应正文

### Requirement: 学习者模型必须使用正式知识坐标

学习者模型 MUST 将课程目标的正式事实投影到正式知识 ID，并继续区分阅读、掌握、自评和推断。个人状态 MUST NOT 改写学科知识库或教学标准库。

#### Scenario: 一个课程目标关联多个细知识点

- **WHEN** 目标产生阅读和正式练习证据
- **THEN** 模型 MUST 在目标层保留原始事实
- **AND** MUST 将相关知识 ID 和聚合状态作为可审计投影返回

## ADDED Requirements

### Requirement: 统一知识库必须包含知识能力易错与提升

系统 MUST 在同一个版本化知识库包中保存正式知识节点、能力点、易错点和提升点。能力点 MUST 归属于一个主知识节点；易错点与提升点 MUST 归属于一个能力点。系统 MUST NOT 为能力、易错和提升建立独立于知识库的版本、真源或学生入口。

#### Scenario: 查询高斯消元知识

- **WHEN** 系统读取高斯消元相关知识
- **THEN** 同一知识库响应 MUST 返回正式知识路径及其能力点
- **AND** 每个能力点下 MUST 分组返回所属易错点与提升点
- **AND** 响应 MUST NOT 再要求客户端加载平行教学标准库

### Requirement: 统一知识库只辅助 AI 不限制 AI

AI MUST 先依据当前课程、任务和证据形成判断，再使用统一知识库中的知识、能力与易错条目统一命名、路径和颗粒度。系统 MUST 允许命中、部分命中和未命中，不得为了填入正式 ID 强迫错误映射。

#### Scenario: 当前错误未被知识库覆盖

- **WHEN** AI 有充分证据发现真实问题但知识库没有对应易错条目
- **THEN** AI MUST 保留该问题并标记为部分命中或未命中
- **AND** 系统 MUST NOT 伪造正式能力或易错 ID

### Requirement: 学生端知识视图必须显示统一父子结构与课程覆盖

学生端 MUST 展示学科语义路径，并叠加当前课程正文、题目、能力、易错、提升和掌握证据。学生端 MUST NOT 用课程、章节和小节重新构造知识树，也 MUST NOT 提供独立教学标准库入口。

#### Scenario: 学生从线性代数课程打开知识库

- **WHEN** 学生打开当前课程知识入口并选择一个知识点
- **THEN** 学生 MUST 看到该知识点下的能力点
- **AND** MUST 在每个能力点下看到对应易错点与提升点
- **AND** MUST 能从同一详情返回当前课程正文

### Requirement: 学习者模型必须使用统一知识坐标

学习者模型 MUST 将正式事实投影到同一知识库中的知识与能力 ID，并只将经过诊断确认的易错条目投影为个人信号。个人状态 MUST NOT 改写正式知识、能力、易错或提升条目。

#### Scenario: 正式练习关联知识能力与易错候选

- **WHEN** 学生提交一项绑定知识、能力和候选易错的正式练习
- **THEN** 作答事实 MUST 保留这些正式引用
- **AND** 一次失败 MUST NOT 直接将候选易错标记为已确认
- **AND** 只有诊断或多次独立证据 MUST 才能形成可审计的易错信号

### Requirement: 课程生产必须消费统一知识库

课程生成 MUST 先形成课程局部知识要求，再与统一知识库匹配，并使用命中知识下的能力、易错和提升切片生成课程目标、正文、题目、诊断和补救资产。模型 MUST NOT 自造正式知识库条目 ID。

#### Scenario: 生成线性代数课程

- **WHEN** 课程局部要求精确命中高斯消元知识
- **THEN** 生成链 MUST 读取高斯消元下的能力、易错和提升条目
- **AND** 正式题目 MUST 保存目标知识与能力引用
- **AND** 无法确认的课程表达 MUST 保持待归一候选

### Requirement: 学习证据必须贯穿统一知识引用

正式题目、作答、学习事件、诊断案例和补救会话 MUST 在各自事实边界内保留可验证的知识、能力与易错引用。引用传播 MUST NOT 把候选易错自动升级为确认结论。

#### Scenario: 两次失败后进入诊断

- **WHEN** 同一目标出现满足门槛的独立失败并创建诊断案例
- **THEN** 诊断案例 MUST 保留来源题目的知识与能力引用
- **AND** 候选易错 MUST 在诊断假设被独立证据确认后才成为确认引用
- **AND** AI 老师 MUST 能读取该确认结果及其证据来源
