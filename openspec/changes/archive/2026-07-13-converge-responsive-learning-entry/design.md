# 设计：并行工具之上的轻量恢复层

## 1. 产品边界

学习页保留多个并行入口，系统不替用户规定下一步。恢复提示只回答“刚才有什么明确没有做完”，不回答“现在最应该做什么”。

```text
稳定并行入口：目录 / 练习 / 记录 / 图谱 / AI
                ↑
可忽略恢复提示：继续上次课程 / 继续未完成练习
```

## 2. 恢复摘要

`GET /api/courses` 在现有课程摘要上增加可选 `resume`：

- `kind`：`reading` 或已有正式任务类型。
- `status`：学习快照中的任务状态。
- `node_id / node_name`：恢复目标。
- `activity_at`：用于在课程库选择最近一项。

该摘要直接来自 `LearningSnapshot`，不建立第二个状态真源，不计算课程进度、待复习数量或 AI 推荐。

## 3. 学习页动作过滤

后端 continuation 投影继续存在并服务旧链路，但 UI 只将 `resume_*` 动作提升为恢复提示。其他动作仍可通过目录、正文任务块和并行工具完成，不再通过底部绿色按钮制造唯一顺序。

AI 可以由用户主动打开，但本变更移除“解释唯一下一步”入口，不新增主动建议。

## 4. 响应式与视觉收敛

- 移动课程库的生成任务和导入按钮压缩为带 tooltip 的图标控制，新建课程保留文字主按钮，三者保持单行且可收缩。
- 移动学习页底栏使用五个等权工具入口；恢复提示在其上方短暂出现，不替代工具栏。
- Markdown 公式、表格、代码块只允许自身横向滚动，页面级容器不得横向滚动。
- AI 默认收起；桌面正文获得完整可读宽度。
- 紫色只用于品牌与主操作，绿色只用于确定的恢复动作，警告与错误使用既有语义 token。

## 5. 兼容与降级

- 没有快照或快照字段不完整时不显示恢复卡，课程仍可正常打开。
- 恢复摘要加载失败不影响课程列表。
- 已有 continuation API、任务执行和 AI Store 不改协议。
