# 设计：旧体验、新内核的学习现场

## 1. 已确认的事实

### 1.1 3 月体验基线

参考点 `9ae4cc7` 已确认：

- `CourseView` 长期保留左课程树和中间连续正文，没有顶部课程模式栏。
- 章节内“题目测试”先打开配置，再以大弹窗完成单题作答。
- 统计、笔记、知识图谱、图片和设置均临时覆盖，不替换正文现场。
- 正文选区把文本和节点上下文交给右侧 AI，不把正式任务塞进聊天。

本次继承空间语义、进入方式和返回方式，不照抄旧版临时题目真源、装饰效果和脆弱布局代码。

### 1.2 当前工程基础

已验证：

- 前端生产构建通过。
- 学习记录、正式练习、进度、运行时和 AI 老师协议的 37 项后端测试通过。
- `PracticeAttempt` 已支持草稿、冲突、提示、AI 支持、提交、重试与诊断补救。
- `LearningRecord` 已支持修订、语义块锚点、迁移状态和归档。
- AI 老师已支持统一会话、来源、提案、幂等回执和撤销。

仍缺失：

- 当前 `/` 与 `/course/:courseId` 都渲染 `CourseView`，没有独立课程库。
- `ContentArea` 主要渲染整段 Markdown，`content_blocks` 只用于锚点，并非真正的可组合课程流。
- `LearningRuntime` 只返回摘要和下一步，没有 `AdaptiveBlock`。
- 手写笔记只保存到块级锚点；同一块重复句子可能错位。
- 锚定 AI 问答不会在首个完整回答后自动幂等沉淀。

## 2. 目标信息架构

```text
/courses
└── CourseLibraryView
    ├── 课程架
    ├── 新建/导入流程
    └── 每门课程自身的生成状态与继续动作

/course/:courseId/learn/:nodeId?
└── LearningShell
    ├── CourseNavigator
    ├── LearningMain
    │   ├── LearningContinuityStrip
    │   └── LearningFlow
    │       ├── CourseBlock
    │       ├── TaskLauncherBlock
    │       ├── InlineLearningRecordBlock
    │       └── AdaptiveBlock
    ├── AITeacherRail
    └── ToolOverlayHost
        ├── TaskOverlay
        ├── LearningRecordsOverlay
        ├── StatsOverlay
        └── KnowledgeGraphOverlay
```

课程流只是投影，不是新真源：

```text
当前课程 content_blocks / Markdown 兼容适配
+ 正式学习资产引用
+ LearningRecord 投影
+ LearningRuntime 临时适配块
→ composeLearningFlow()
→ 有序 LearningFlowItem[]
```

## 3. 组件处理清单

| 当前组件或能力 | 处理 | 目标归位 |

| `CourseTree` | 改造 | 只负责课程结构、节点状态、记录与图谱入口 |
| `ContentArea` | 拆分 | 滚动/虚拟化、选区、记录、工具和正文渲染分离 |
| `CourseNode` | 换壳复用 | 保留节点层级语义，内部改为课程流渲染 |
| `MarkdownDocumentEditor` | 保留兼容 | 作为 rich_text/raw_markdown 块渲染器，不再拥有整节课程 |
| `PracticeWorkspace` | 保留能力 | 去掉独立页面外壳，嵌入 `TaskOverlay` |
| `SideAIPanel` | 改造 | 成为唯一 `AITeacherRail`，宽屏常驻、窄屏覆盖 |
| `NotesPanel` | 换壳复用 | 作为按需打开的学习记录总览工具 |
| `CourseWorkspace` | 退役外壳 | 删除平级模式；内部可复用逻辑迁回对应领域组件 |
| `SmartBar` | 旧六按钮实现退役 | 桌面改为正文工作区内的记录/正式练习/统计/图谱工具坞；移动端另建低密度“目录/正文/记录/AI”导航 |
| `LearningContinuityBar` | 学习页常驻外壳退役 | 保留连续性投影；唯一下一步进入工具坞，解释进入 AI，章节结果进入统计工具 |

| 生成 WebSocket/轮询 | 保留 | 只在对应课程项和创建流程显示 |

## 4. 界面状态与领域状态

前端只新增一个轻量 `learningUi` 协调 Store：

```text
activeOverlay
overlaySourceRef
returnAnchor
aiRailState
mobileSurface
expandedInlineRecordId
recordFilters
```

它不得保存当前目标、掌握、正式任务状态、课程版本业务结论或 AI 推荐下一步。写操作固定为：

```text
领域 API 写入
→ 返回正式对象/回执
→ 刷新 LearningRuntime
→ composeLearningFlow 重新投影
→ 正文与 AI 上下文共同刷新
```

## 5. 正文任务工具

`TaskLauncherBlock` 只保存任务引用和展示摘要，不复制题目。点击时：

1. 保存 `node_id + block_id + scroll offset + task_ref`。
2. `learningUi.openOverlay('task', sourceRef)`。
3. `PracticeWorkspace` 根据 `task_ref` 精确恢复 Attempt 或诊断任务。
4. AI 老师上下文增加当前 `task_ref`，但仍是同一个右栏。
5. 关闭、提交或按 `Esc` 后先刷新运行时，再恢复锚点；恢复失败时回到来源块而非页面顶部。

桌面普通题目覆盖中间主区；代码实验和全图探索才允许全屏。移动端统一全屏。重型工具同一时刻最多一个。

## 6. 行内学习记录

### 6.1 精确锚点

沿用现有 `anchor`，补充：

```text
block_id
block_revision_id
text_position: { start, end }
text_quote: { exact, prefix, suffix }
```

解析顺序为精确位置、唯一上下文引用、当前块唯一引用、块级待确认。不得保存像素坐标。

### 6.2 手写便签

选区点击“笔记”后，原文附近直接出现 `InlineLearningRecordBlock` 编辑态。输入短暂停顿、失焦或快捷键触发保存；创建失败时保留本地草稿并明确显示，不伪装成功。

短纯文本便签在宽屏可环绕；长笔记、公式、代码和移动端全部占完整正文宽度。同一锚点最多展开一个环绕便签。

### 6.3 锚定 AI 问答

仅当入口为 `selection` 且存在可靠 `content_anchor` 时，首个完整回答触发 `upsert_anchored_qa_record`：

- 幂等键由用户、课程、会话根问题和锚点组成。
- `metadata.record_subtype=ai_qa`，保存会话和消息引用。
- 后续追问更新同一记录，不新增第二张块。
- 摘要失败不影响 AI 回答；记录显示可重试，不保存半段流式内容。
- 普通聊天、AI 主动提醒、无锚点提问和中间输出不自动保存。

## 7. 临时适配块

`LearningRuntime` 可返回：

```text
adaptive_block_id
anchor
kind
role
payload
reason_code
evidence_refs
status
expires_at
```

它只表达低风险解释、反例、过渡或非正式理解检查。用户可跳过、收起或反馈；同一锚点同一时刻最多一个。正式课程、正式题目、Attempt 和掌握事实不得由该块直接修改。

## 8. 视觉基线

视觉源不是对 3 月版本的抽象概括，而是参考点 `9ae4cc7` 的实际组件和样式。后续不得重新发明一套“更现代”的界面：

- 顶栏、目录、正文和 AI 侧栏直接继承 3 月版的浅色渐变背景、玻璃外壳、柔和阴影、16-24px 分级主容器圆角和 indigo 品牌色。
- 顶栏继承旧版紧凑品牌标识、轻巧图标按钮和柔和反馈，同时采用新版更规整的三段对齐与课程上下文密度；不得做成无品牌特征的后台工具条。
- 目录继承旧版书本图标、虚线章节脉络、圆点节点和柔和选中面，同时保留新版搜索、紧凑行高、阅读状态和掌握状态。
- 课程章节直接继承深色 `Chapter` 徽标、浅色编号、紫色分隔线、白色大章节卡和左侧章节脉络；不另造新的卡片语言。
- 正文教学块之间不用整行横线切割，依靠留白、角色标签和标题层级建立节奏；装饰性 `hr` 不渲染。`小结` 等收束型标签和标题必须高于普通元数据层级。
- 绿色只表示完成或验证，amber 表示提醒，red 表示阻断；功能状态不得改变品牌主色。
- 当前版本新增的学习进度、真实练习入口、AI 老师、课程架和响应式只嵌入上述结构，不反向改造旧版阅读节奏。
- 允许清理嵌套模糊、重复渐变、重复 token 和影响滚动性能的滤镜，但清理后必须保持同视口视觉身份不变。
- 禁止卡片套卡片和无功能装饰；状态标签才使用胶囊，功能按钮使用既有图标与尺寸。
- 每次迁移先对照同视口旧版截图和源码，再由人工确认后扩大范围。

## 9. 响应式规则

| 视口 | 目录 | 正文 | AI 老师 | 临时工具 |
| --- | --- | --- | --- | --- |
| `>=1280` | 248-280px 固定 | 最低 640px 可读宽度 | 320-360px 固定 | 中间覆盖或必要时全屏 |
| `1024-1279` | 固定 | 主区优先 | 覆盖侧板 | 中间覆盖 |
| `768-1023` | 覆盖抽屉 | 占满 | 覆盖侧板，与目录互斥 | 大覆盖层 |
| `<768` | 全屏/底部入口 | 单主区 | 全屏 | 全屏 |

移动底部导航只允许“目录、正文、记录、AI”。桌面恢复正文工作区内的低频工具坞，但不恢复旧 `SmartBar` 的 AI 出题、本地错题和重复 AI 入口；工具坞中的练习、统计与图谱均调用现有正式能力。

## 10. 分阶段执行与人工门禁

### A. 骨架样板

建立独立课程库路由、学习外壳、统一 token 和一门真实课程的静态课程流。只验证空间、密度和响应式，不接全部写操作。

**门禁 A**：用户确认左中右、正文宽度、任务入口、行内记录和 AI 常驻方式。

### B. 第一条完整学习链

接通“选择目标 -> 阅读 -> 选区问 AI -> 问答沉淀 -> 行内便签 -> 任务覆盖层 -> 正式作答 -> 诊断补救 -> 返回原文”。

**门禁 B**：真实后端、真实课程、真实刷新与断点恢复均通过。

### C. 退役重复入口

删除平级模式栏、固定笔记栏、旧六按钮 `SmartBar`、AI 浮球和学生生产界面；新增边界清晰的桌面学习工具坞并保留必要兼容适配器。

**门禁 C**：旧入口无残留，领域能力仍可从新位置到达。

### D. 扩展与收口

迁移课程库创建流程、记录总览、统计、图谱和剩余弹层；实现 `AdaptiveBlock`，清理视觉与布局技术债。

**门禁 D**：四视口中英文真实浏览器验收和全链路回归通过后归档。

## 11. 风险控制

- 不在第一步删除 `CourseWorkspace` 文件；先迁出可复用逻辑并建立到达性测试，再退役外壳。
- 不让 `LearningFlow` 保存第二份排序真源；兼容期的任务和记录插入必须由稳定锚点与确定性规则计算。
- 课程首页只做用户已确认的课程架改造；学习页仍以 3 月版外壳为基线，每阶段保持可回滚。
- 不依赖截图单独宣称完成；同视口参考与实现并排比较，并验证实际点击、恢复、草稿和上下文。
- 运行时数据文件继续隔离，不纳入 UI 改造提交。
