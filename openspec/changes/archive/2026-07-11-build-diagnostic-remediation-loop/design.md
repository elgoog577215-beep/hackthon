# 设计

## 1. 真源与边界

- `PracticeAttempt` 保存所有真实作答；任务用途不同，但不建立第二套草稿、提交和评分系统。
- `DiagnosticCase` 保存当前诊断流程，`DiagnosticHypothesis` 只保存待验证或已决策假设。
- `RemediationSession` 保存当前补救与返回位置，`LearningEvent` 追加保存过程事实。
- 学习者模型只读取已确认且满足跨情境门槛的模式；单个案例不直接成为稳定画像。

## 2. 统一任务

`AssessmentTaskRevision` 的用途固定为 `course_practice / diagnostic_probe / remediation_guided / remediation_validation`。旧正式题目通过读取投影成为统一任务；案例任务使用不可变修订。所有 Attempt 保存 `task_revision_id`、用途、来源和案例关联，旧 `question_revision_id` 保持兼容。

辨别题和引导练习不得证明掌握。独立复验只有在当前目标与标准修订有效、质量门通过、答案未泄露且支持等级为零时，才可以成为掌握证据。

## 3. 诊断决策

首次普通失败只给反馈和重试。当前版本高置信正式错误重复出现，或独立复验再次失败时，才开启案例。每个案例最多三个候选假设和三道辨别题。

AI 可以提出候选假设和长尾辨别题，但确认、排除和未确定由版本化规则根据支持证据、反对证据、竞争假设和任务有效性决定。低置信评分、二三级提示、完整解析和旧迁移历史不能确认错因。

## 4. 最小补救与复验

补救来源优先为课程内容锚点、正式补救资产、最后才是 AI 临时内容。每个补救单元只处理一个确认错因，并包含最短解释、一个对比或示例和一个主动操作。

完成补救后必须使用未展示的新题独立复验。通过后关闭案例并返回原路线；相同错误会重开假设并更换补救；新错误重新进入候选；低置信结果等待评阅。两次补救失败后升级为交互诊断，不无限生成题目。

## 5. 前端

`PracticeWorkspace` 内部增加 `practice / diagnostic / remediation / validation / resolved` 阶段。每个阶段只显示一个主要动作，支持刷新和跨设备恢复。请求 AI 帮助后当前辨别或复验结果降级，系统明确安排新的独立任务。

## 6. 旧链清理

删除或降级基于 `quiz_score`、旧错题数量、停留时间和单次失败直接生成稳定薄弱点或阻止继续学习的规则。旧错题接口不得抢占导师主行动，统计只能描述 Attempt 结果，不能替代诊断决策。
