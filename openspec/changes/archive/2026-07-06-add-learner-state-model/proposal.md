# 添加学习者状态模型

## Why

统一学习事件账本已经能记录用户提问、AI 回答、标注保存和导师学习结果，但事件本身只是事实流。后续教学决策层需要一个可查询、可解释的状态层，把事实事件聚合成节点活跃度、近期活动、薄弱点候选、学习偏好证据和当前风险。

## What Changes

- 新增轻量 `LearnerState` 聚合模块，从 `LearningEvent` 读取事件并生成结构化状态。
- 支持按默认用户、课程、节点过滤状态。
- 明确区分事实证据和系统推断。
- 为 AI 助手 prompt 追加学习者状态摘要，但不替代现有笔记、错题和画像。

## Out of Scope

- 不实现教学决策层。
- 不新增数据库或迁移。
- 不改前端 UI。
- 不把状态模型当成静态画像。
