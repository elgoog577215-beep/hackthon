## ADDED Requirements

### Requirement: 学习事件必须可聚合为学习者状态

系统 MUST 提供轻量 `LearnerState` 聚合能力，从统一学习事件账本读取事实事件，并生成可查询、可解释、可后续教学决策使用的结构化学习者状态。状态模型 MUST 区分事实证据和系统推断，MUST 使用现有文件存储和默认用户兼容模式。

#### Scenario: 从事件聚合节点状态

- **WHEN** 系统读取指定用户、课程或节点的学习事件
- **THEN** 系统 MUST 返回节点级交互次数、提问次数、笔记数量、错题数量和最近答题结果摘要
- **AND** 状态 MUST 保留近期活动作为事实证据

#### Scenario: 生成薄弱点和风险提示

- **WHEN** 某节点出现错题、低正确率或重复提问
- **THEN** 系统 SHOULD 生成薄弱点候选
- **AND** 系统 SHOULD 生成当前风险提示
- **AND** 这些输出 MUST 标记为系统推断，而不是事实事件

#### Scenario: AI 助手读取学习者状态摘要

- **WHEN** AI 助手构造问答 prompt
- **THEN** prompt SHOULD 包含学习者状态摘要
- **AND** 该摘要 MUST 追加在现有学习者上下文之后
- **AND** 不得替代已有笔记、错题、画像和导师记忆
