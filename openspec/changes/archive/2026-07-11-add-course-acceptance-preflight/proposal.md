# 变更：增加课程全链路只读预检

## Why

课程使用前七步已有独立测试和局部真实验收，但当前课程目录中的课程普遍没有正式练习资产，仍能依靠读取层投影目标和内容块正常打开。如果直接进入完整学习链，直到练习、诊断或章节收束才会发现生成产物不满足课程使用契约，容易把兼容降级误判为全链路通过。

第 8.2 步需要一个确定性、只读、机器可读的预检能力，在任何学习状态写入前检查版本、目标、内容、练习、补救和推进六项契约，并明确区分标准课程、阅读型课程和旧课程兼容口径。

## What Changes

- 新增纯函数课程契约检查器，输出稳定 schema、六项契约结果、阻断项、警告和资产统计。
- 新增只读课程目录扫描器，显式报告损坏 JSON，不触发恢复、迁移、编译或保存。
- 新增单课程和全目录只读 API，默认采用严格 `standard` 口径。
- 新增命令行入口，支持 JSON 输出、按课程过滤和阻断时非零退出。
- 为标准、阅读型、兼容、损坏数据和真实目录扫描补充测试。
- 将真实预检结果回写课程使用全景，作为第 8.3 标准样本生成的前置证据。

## 影响范围

- 后端：新增课程验收检查器和只读 API；不修改课程生成、课程存储和学习状态。
- 工具：新增开发者命令行预检入口。
- 文档：更新全链路验收矩阵、产品蓝图和项目决策。
- 数据：无迁移、无写入；损坏文件只报告，不自动恢复。
