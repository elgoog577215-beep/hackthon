# 设计：课程全链路只读预检

## 1. 设计目标

预检回答两个问题：当前课程是否具备进入完整学习链的最低契约；如果不能，缺口属于哪个课程使用契约。它不回答教学内容是否优秀，也不替代后续真实行为验收。

## 2. 验收口径

- `standard`：六项契约全部必需，缺失正式题目、补救资产或推进契约时阻断。
- `reading_only`：版本、目标、内容和推进必需；练习与补救必须被生成计划明确关闭，结果为降级而非失败。
- `compatibility`：只要求旧课程能形成目标和内容投影；版本、正式练习、补救和推进缺失均明确降级。
- `auto`：只有生成计划显式声明 `reading_only_degraded` 时选择 `reading_only`，否则始终选择 `standard`，不根据资产缺失自动降级。

## 3. 数据路径

检查器先读取持久课程原文，再在深拷贝上调用现有目标与内容投影。报告分别记录持久字段和兼容投影结果。正式题目、量规、补救资产、版本和章节推进契约只认持久化数据，不调用资产编译器现场补齐。

目录扫描器直接解析主课程 JSON 文件，跳过旧 `.vN.json` 快照。无效 JSON 形成 `storage_integrity` 阻断报告，不能调用带自动恢复能力的存储验证方法。

## 4. 报告结构

单课程报告包含 schema、生成时间、课程身份、请求口径、实际口径、总体状态、六项契约、阻断项、警告和统计。每项检查包含稳定检查 ID、严重级别、预期、实际和中文说明。

目录报告包含汇总计数、逐课程报告和损坏文件报告。CLI 可选择在存在阻断项时返回退出码 2，便于后续 CI 使用。

## 5. 边界

- 预检不得创建初始课程版本、编译资产、迁移旧记录、恢复损坏文件或写入学习事件。
- 预检不得把兼容投影的目标和内容误报为生成器已经持久化完整资产。
- API 只提供诊断结果，不成为前端新的学习状态真源。
- 第 8.3 才根据报告修复生成链并创建标准样本。
