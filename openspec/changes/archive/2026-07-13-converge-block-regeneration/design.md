# 设计：候选先行，课程命令落盘

## 1. 唯一流程

```text
正文 CourseBlock 的轻量 AI 入口
-> 右侧 AI 老师收集改进要求
-> 校验当前 document revision 与 block revision
-> 读取目标块、所属章节和相邻块上下文
-> 生成候选内容
-> 确定性质量门
-> 必要时自动修订一次
-> 保存独立候选，当前课程不变
-> 用户预览并确认
-> CourseCommandService.replace_block
-> 原子保存 CourseDocument 与幂等回执
-> 学习页原位刷新
```

候选不是第二份课程真源。它只保存一次待确认修改所需的原始修订、建议载荷、质量报告和处理状态；当前课程仍只认 `CourseDocument`。

## 2. 候选契约

每个候选至少保存：

- `candidate_id / request_id / course_id / block_id / section_id`
- 创建时的 `expected_document_revision / expected_block_revision`
- 用户指令和动作类型
- 原载荷摘要与建议载荷
- 生成尝试次数、质量门和问题列表
- `ready / quality_failed / applied / rejected / stale` 状态
- 应用后的课程命令回执

相同 `request_id` 重试时返回原候选，避免重复调用模型。候选文件按 ID 隔离并原子写入，不写入课程文件。

## 3. 不变量

局部重新生成只替换 `CourseBlock.payload`，因此以下字段由当前正式块保留：

- `block_id`
- `section_id / parent_group_id / position`
- `kind / role / status`
- `asset_refs / objective_refs / concept_refs / evidence_refs`
- `visibility_rule`

候选生成不得整节重写，也不得通过旧节点投影保存课程。若用户真正要求改变块类型、结构或正式引用，应另行建立对应领域命令，不借局部重写绕过边界。

## 4. 质量门

质量门至少检查：

- 输出存在有效正文，且不是原文原样返回。
- 输出没有错误占位、模型拒答或异常标记。
- 输出长度处于块级安全范围。
- 原文包含代码围栏时，候选必须保留有效且闭合的代码围栏。
- 建议载荷仍保留原有非正文载荷字段。

首次未通过时，把失败项作为反馈重新生成一次。第二次仍失败时保存 `quality_failed` 候选及报告，禁止应用。质量门只保护明确可验证的不变量，不把难以解释的风格偏好伪装成确定事实。

## 5. 并发、幂等与冲突

创建候选时先校验客户端看到的课程修订和块修订，防止基于过期内容支付模型成本。确认时再次由课程命令校验同一组修订：

- 修订一致：应用并更新块修订与课程修订。
- 任一修订不一致：当前课程不变，候选标记为 `stale`，前端提示重新生成。
- 同一候选重复确认：命令 ID 固定为候选 ID，返回原回执。

## 6. 前端归位

正文块标题旁只出现一个悬浮可见的 AI 改进图标，不增加常驻工具条。点击后打开既有右侧 AI 老师并带入目标块；右栏使用同一视觉语法显示：目标上下文、改进要求、生成状态、候选正文、质量状态以及确认或放弃。

应用成功后重新读取正式课程文档，并按原 `block_id` 保持阅读位置。窄屏继续使用现有 AI 覆盖层，不新增移动端页面。

## 7. 旧链边界

未迁移历史课程可暂时保留旧节点块接口以避免破坏读取兼容。规范课程调用旧接口时必须在模型调用前返回冲突和新入口信息；前端规范课程只使用新候选接口。旧 `CourseVersion` 整课候选链不参与本流程。
