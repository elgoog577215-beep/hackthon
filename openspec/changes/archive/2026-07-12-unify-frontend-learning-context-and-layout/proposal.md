# 变更：统一前端学习范围与窄屏布局

## 为什么

第 8.4 至 8.5C 已经证明课程学习后端主链可用，但产品一致性审计发现前端仍有四个 P0 阻断：连续性条读取 `LearningRuntime.current_objective`，练习与掌握却只读取旧 `courseStore.currentNode`；768 至 1024px 同时保留目录、正文和固定笔记栏，正文会缩到不可读；练习与掌握没有明确展示当前作用域；390px 主导航隐藏全部文字，只剩难以辨认的图标。

## 改什么

- 以前端现有 `LearningRuntime.current_objective` 作为未显式选择节点时的正式范围回退，不新增范围状态。
- 让练习和掌握明确展示当前目标、全课程或综合检测作用域，并保持接口参数与页面标签一致。
- 在 768 至 1024px 使用抽屉式课程目录并默认收起固定笔记栏，保证正文可读宽度。
- 在移动端保留课程模式文字，使用可横向滚动的紧凑导航，不再只显示无文字图标。
- 增加范围解析和关键文案测试，完成中英文、桌面、窄桌面和移动端验收。

## 不做什么

- 不新增学习状态、推荐状态或页面级范围真源。
- 不在本变更拆分课程库、学习工作区和课程工作室路由。
- 不重做整站视觉系统、创建课程流程或旧阅读器。
- 不修改后端课程生成、练习、诊断和掌握领域逻辑。

## 影响

- 前端：课程页布局、课程工作区范围、练习/掌握标题、移动端导航。
- 规范：LearningRuntime 前端范围消费和窄屏可读性。
- 验收：390px、789px、1024px、1440px，中英文和当前目标恢复。
