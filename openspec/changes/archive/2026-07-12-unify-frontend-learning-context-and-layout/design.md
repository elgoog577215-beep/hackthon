# 设计：前端学习范围与窄屏布局收口

## 1. 范围真源

正式学习范围按以下顺序解析：

1. 用户在当前课程中显式选择的有效课程节点。
2. `LearningRuntime.continuation.current_objective` 指向的正式节点。
3. 两者都不存在时才使用全课程范围。

该解析只生成组件展示参数，不持久化第二份状态。初次加载课程时，运行时目标可以帮助工作区恢复；用户主动浏览其他节点后，前端尊重显式选择。

## 2. 作用域展示

练习作用域固定为：

- `node`：显示“当前目标”与节点名称，只请求该节点题目。
- `all`：显示“全课程”，不携带节点参数。
- `final`：显示“综合检测”，不把当前节点名称伪装成综合检测范围。

掌握工作区只展示当前目标范围。缺少可解析目标时，页面显示明确的全课程回退，不再使用“章节掌握”标题制造错误承诺。

## 3. 响应式布局

- `< 768px`：课程目录为移动抽屉。
- `768px - 1023px`：课程目录同样使用覆盖式抽屉；固定笔记栏默认收起，避免三栏并存。
- `1024px - 1279px`：允许持久目录，但固定笔记栏继续收起。
- `>= 1280px`：允许持久目录和按用户状态显示的固定笔记栏。

布局门禁不是“无横向溢出”即可，而是中央正文必须保持可读宽度。

## 4. 移动端导航

课程模式导航保留图标和短文字，允许横向滚动；每个按钮保持稳定高度与触控宽度。不得通过隐藏全部标签来压缩六个并列入口。

## 5. 验收

1. 无旧节点选择时，练习和掌握使用运行时当前目标。
2. 用户选择其他节点后，工作区使用该节点，运行时仍保持唯一正式学习状态。
3. node/all/final 三种练习范围的请求参数和标题一致。
4. 789px 下正文不再被目录与笔记栏夹成窄列。
5. 390px 下模式名称可见，导航可滚动且不覆盖连续性条。
6. 中文与英文作用域文案完整。
