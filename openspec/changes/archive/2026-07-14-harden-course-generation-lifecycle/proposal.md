# 变更：收紧课程生成任务全生命周期

## Why

课程正常生成已经能够从创建任务走到质量检查和正式发布，但异常路径仍没有形成同一个生命周期：取消任务只删除任务记录，可能留下空课程外壳和孤立工作区；删除课程只删除课程文件，可能让后台任务继续写入；暂停、节点控制和清理失败任务在对象不存在或状态不允许时仍可能返回成功；创建任务中途失败也缺少补偿清理。

这些问题会把一个课程生成请求拆成彼此失联的任务、课程、工作区和前端缓存。正常路径越强，异常路径残留的数据越多，最终表现为 0 节点课程、任务中心与课程库不一致、删除后重新出现、按钮显示成功但实际没有执行。

## What Changes

1. 将课程生成创建、取消、删除、失败清理定义为同一个后端生命周期，由 `TaskManager` 统一编排任务、运行协程、生成工作区、候选版本和初始课程外壳。
2. 为创建过程增加补偿清理：任一步失败都撤销本次请求已经创建、但尚未发布的对象。
3. 取消初始课程生成时，停止并等待后台协程，删除任务专属工作区和未发布课程外壳；已正式发布的课程内容必须保留。
4. 删除课程时先终止该课程关联的生成任务，再删除正式课程，防止后台任务在删除后继续写入。
5. 暂停、节点控制、删除任务和清理失败任务返回真实结果；不存在或终态任务不得伪装成操作成功。
6. 课程主题在任何持久化操作前统一去除首尾空白并校验，拒绝空白主题。
7. 前端课程库、任务中心和生成进度只投影后端真实状态；取消、删除或发现新任务后立即对账课程列表与本地缓存。
8. 修正不稳定的剩余时间展示：样本不足时显示计算中，避免根据单次并发进度跳变给出虚假精确值。
9. 明确模型调用超时与重试所有权：SDK 不再暗中叠加长时间重试，业务层负责有限重试；流式 provider 错误必须作为失败进入恢复链，不能写进课程正文。
10. 将质量闸门拆成“发布阻断”和“优化建议”：空正文、无效证据、必用资料缺失和资产阻断继续禁止发布；少量难度或迁移建议允许课程带警告发布。
11. 为首次课程创建增加端到端请求幂等；同一提交的网络重试只返回原任务和课程。
12. 课程列表区分正式课程、真实任务承接的生成外壳和失联历史外壳，后者保留审计但不再展示为 0 节点课程。

## Out of Scope

- 不重写已经通过真实课程验证的 prompt、学科模板、知识树、正文生成和质量评分算法。
- 不增加第二套调度器、任务身份或前端生成真源。
- 不在本变更中迁移历史损坏课程数据；只增加可发现、可清理和不再继续制造同类残留的机制。
- 不改变模型候选列表、prompt、采样参数和教学内容质量策略；本轮只收紧等待上限、重试归属和错误传播。

## Impact

- 后端：`TaskManager` 生命周期、课程删除路由、生成请求校验、工作区与版本仓库清理能力、模型 provider 运行边界。
- 前端：课程库删除入口、任务取消后的对账、跨标签页发现任务后的列表刷新、剩余时间文案。
- 测试：创建补偿、取消、课程删除、暂停、节点控制、失败清理、空白输入和前端状态同步回归。

## Success Criteria

- 创建、暂停、恢复、取消、失败、清理、删除和正式完成均能得到真实且一致的任务状态。
- 取消未发布的新课程后，不再残留任务、工作区、候选版本或 0 节点课程外壳。
- 删除课程后，不存在仍可继续写入该课程的活动任务。
- 已发布课程被清理任务记录时，正式课程文档不受影响。
- provider 长时间无响应会在明确上限内进入现有节点重试与失败恢复，错误文本不会成为课程正文。
- 全部正文和关键契约已经满足时，非阻断质量建议不会再让用户只得到 0 节点课程外壳。
- 浏览器刷新、退出重进和跨标签页场景下，课程库与任务中心无需手动刷新即可对账。
- 同一请求号并发或重复提交只产生一个任务、一个工作区和一门课程。
- 历史失联外壳不会进入课程架，正式课节数量只统计二级可学习节点。
- 后端完整测试、前端完整测试、生产构建、OpenSpec 严格校验和真实纵向验收全部通过。
