## ADDED Requirements

### Requirement: 学科知识库必须独立于具体课程

系统 MUST 以版本化学科包保存正式知识节点和关系。正式知识 ID MUST NOT 包含课程 ID，课程、章节和小节 MUST NOT 成为学科知识节点。

#### Scenario: 两门课程引用同一知识

- **WHEN** 两门线性代数课程都覆盖高斯消元法
- **THEN** 两门课程 MUST 引用同一个正式知识 ID
- **AND** 两门课程 MUST 分别保存自己的课程映射 ID 和正文位置

### Requirement: 学科知识库必须收束到细知识点

学科知识库 MUST 使用学科、领域、主题、概念和细知识点表达语义层级。概念 MAY 聚合多个细知识点；所有叶子 MUST 为细知识点。

#### Scenario: 宽概念展开为细知识

- **WHEN** 学科包包含“初等行变换与阶梯形”概念
- **THEN** 该概念 MUST 展开为合法行变换、解集保持、主元与阶梯形、行最简阶梯形等细知识点
- **AND** 课程只需映射概念或其部分细知识点，不得复制学科层级

### Requirement: 正式知识关系必须与课程顺序分离

学科前置、相关、易混、迁移和应用关系 MUST 由学科包保存；课程讲授顺序 MUST 只存在于课程知识映射中。

#### Scenario: 课程调整章节顺序

- **WHEN** 一门课程调整矩阵秩与线性方程组的讲授顺序
- **THEN** 学科知识关系 MUST 保持不变
- **AND** 只有该课程的映射顺序和覆盖关系 MAY 改变

### Requirement: 教学标准库必须区分能力易错与提升

系统 MUST 将能力点、易错点和提升点保存为不同正式对象。能力点 MUST 关联一个主知识节点；易错点 MUST 归属于能力点；提升点 MUST 归属于能力点或知识节点。

#### Scenario: 查询高斯消元教学标准

- **WHEN** 系统读取高斯消元相关标准
- **THEN** 响应 MUST 区分可观察能力、具体易错行为和进阶训练方向
- **AND** 易错点 MUST 包含错误表现、认知原因和辨析或修复方法

### Requirement: 标准库只服务 AI 不限制 AI

AI MUST 先依据当前课程、任务和证据形成判断，再使用标准库统一命名、路径和颗粒度。系统 MUST 允许命中、部分命中和未命中，不得为了填入正式 ID 强迫错误映射。

#### Scenario: 当前错误未被标准库覆盖

- **WHEN** AI 有充分证据发现真实问题但正式标准库没有对应条目
- **THEN** AI MUST 保留该问题并标记为部分命中或未命中
- **AND** 系统 MUST NOT 伪造正式标准 ID

### Requirement: 课程知识映射必须明确正式与待归一内容

课程知识映射 MUST 保存课程局部表达到正式知识的匹配方式、置信度和覆盖细知识点。只有规范名称或正式别名精确命中才能自动成为正式映射；其余内容 MUST 保持待归一。

#### Scenario: 旧课程只有大纲关键点

- **WHEN** 旧课程关键点可精确命中正式名称或别名
- **THEN** 系统 MUST 确定性建立正式映射
- **AND** 无法命中的关键点 MUST 成为待归一候选而不是新正式知识

### Requirement: 学生端知识视图必须显示学科语义与课程覆盖

学生端 MUST 展示学科语义路径，并叠加当前课程正文、题目、掌握标准和误区绑定。学生端 MUST NOT 用课程、章节和小节重新构造知识树。

#### Scenario: 学生从线性代数课程打开知识库

- **WHEN** 学生打开当前课程知识入口
- **THEN** 根节点 MUST 是学科或领域语义节点
- **AND** 学生 MUST 能看到当前课程覆盖标记并跳回相应正文

### Requirement: 学习者模型必须使用正式知识坐标

学习者模型 MUST 将课程目标的正式事实投影到正式知识 ID，并继续区分阅读、掌握、自评和推断。个人状态 MUST NOT 改写学科知识库或教学标准库。

#### Scenario: 一个课程目标关联多个细知识点

- **WHEN** 目标产生阅读和正式练习证据
- **THEN** 模型 MUST 在目标层保留原始事实
- **AND** MUST 将相关知识 ID 和聚合状态作为可审计投影返回
