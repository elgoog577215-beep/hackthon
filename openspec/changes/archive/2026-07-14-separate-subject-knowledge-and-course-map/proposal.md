# 提案：分离学科知识库、教学标准库与课程知识映射

## 背景

当前 `knowledge_tree_v1` 以课程为根，依次复制章节、小节和主题，再把课程大纲中的关键点作为叶子。它虽然解决了正文、题目和误区缺少稳定引用的问题，但本质仍是课程目录的细化投影：同一知识在不同课程中会拥有不同身份，课程顺序被误当成知识关系，旧课程还会生成名为“核心知识”的占位主题。学生端因此看到的是第二份课程目录，学习者模型和 AI 老师也无法积累跨课程知识证据。

Code 平台现有设计证明了更合理的边界：学科知识库负责“学什么”，教学标准库负责“应该会什么、常错什么、怎样提升”，具体课程只负责选择知识、组织顺序和绑定正文。灵知需要吸收这个边界，但做成可加载不同学科包的通用结构，而不是复制信息学硬编码目录。

## 目标

1. 建立独立、版本化、跨课程复用的学科知识库，知识身份不包含课程 ID。
2. 建立挂在学科知识节点下的教学标准库，正式区分能力点、易错点和提升点。
3. 把现有课程五层树降级为课程知识映射，只保存课程节点、正文块和正式知识节点之间的覆盖关系。
4. 让课程生成优先复用已知学科术语；无法精确映射的内容保持待归一状态，不伪装成正式知识。
5. 让题目、掌握标准、误区、AI 上下文和学习者模型共享正式知识 ID 与教学标准 ID。
6. 将学生端入口改为学科知识视图：展示语义路径、当前课程覆盖、教学标准和学习证据，而不是重复课程目录。

## 非目标

- 不在本轮建设完整的多学科内容库；先以线性代数学科包证明通用模型和端到端链路。
- 不把课程生成模型输出直接写入正式学科知识库或教学标准库。
- 不把标准库当作强制答案表；AI 必须允许命中、部分命中和未命中。
- 不把个人掌握状态写回学科知识库；掌握只作为学习者模型覆盖层。
- 不恢复旧力导向图、学生端知识编辑器或 AI 自动覆盖正式知识的旧链路。

## 成功标准

- 当前线性代数课程的知识视图根节点为学科语义节点，不再包含课程、章节或小节节点。
- 两门课程引用同一知识时得到相同正式知识 ID，课程映射拥有独立映射 ID。
- 课程正文、题目、掌握标准和误区只引用存在于正式学科知识库中的知识 ID；未映射内容被明确列为候选。
- AI 上下文能读取当前课程节点关联的知识、能力点、易错点和提升点，并明确这些内容只用于导航、命名和颗粒度校准。
- 学习者模型按正式知识 ID 聚合当前目标的证据，阅读状态与正式掌握继续分离。
- 旧课程无需重新调用 AI 即可生成课程知识映射；旧知识图谱不再作为运行时真源。
- 学生端桌面和移动端均能搜索知识、查看课程覆盖与教学标准，并跳回对应正文。
