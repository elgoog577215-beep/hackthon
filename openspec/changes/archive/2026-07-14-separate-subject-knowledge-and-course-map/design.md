# 设计：知识基础设施三层分离

## 一、核心边界

### 1. 学科知识库

学科知识库是跨课程复用的正式语义真源。节点层级为：

```text
学科 → 领域 → 主题 → 概念 → 细知识点
```

节点保存稳定 ID、规范名称、别名、定义、学习动作、典型问题、来源、状态和版本。包含关系只由父子字段表达；前置、相关、易混、迁移和应用关系独立保存。正式 ID 由学科包决定，不包含课程 ID。

### 2. 教学标准库

教学标准库是知识点以下的教学与诊断层：

```text
知识节点 → 能力点 → 易错点
知识节点或能力点 → 提升点
```

能力点描述可观察行为；易错点描述具体错误表现、认知原因、辨析方法和修复策略；提升点描述进阶目标、练习策略和收益。标准库只给 AI 和教学流程提供候选地图、统一命名和颗粒度，不替代当前学习证据。

### 3. 课程知识映射

课程知识映射保存课程节点、正文块与正式知识节点之间的关系。它可以表达本课程的讲授顺序和覆盖范围，但不得改变正式知识层级或创建另一套正式知识 ID。

每条映射保存：课程节点、局部名称、局部主题、正式知识锚点、覆盖的细知识点、匹配方式、置信度、正文块、目标和证据来源。精确名称或正式别名才能自动成为正式映射；相似但无法确认的内容保持待归一候选。

## 二、线性代数首个学科包

首个学科包覆盖向量与线性组合、矩阵与线性映射、线性方程组、线性空间四个主题。课程大纲中的宽粒度表达映射到“概念”节点，并展开到更细知识点。例如：

```text
矩阵与线性映射
└─ 初等行变换与阶梯形
   ├─ 三类合法初等行变换
   ├─ 行变换保持方程组解集
   ├─ 主元位置与阶梯形条件
   └─ 行最简阶梯形的唯一性
```

课程中的“初等变换与行简化阶梯形”只负责覆盖这个概念，不再成为课程专属正式知识节点。

## 三、生成与迁移

### 新课程

1. 课程主题先匹配学科包。
2. 大纲 prompt 获得相关学科知识摘要，优先使用规范名称和已有别名。
3. 后端对模型输出做确定性映射，不接受模型自造正式 ID。
4. 正文 prompt 接收当前课程节点的知识覆盖和相关教学标准。
5. 无法映射的内容进入课程映射候选，不写入正式库。

### 旧课程

旧课程从现有 `key_points`、`knowledge_structure` 和旧图谱标签确定性生成课程映射。只有精确名称或别名命中才绑定正式知识 ID；其余内容保持待归一。迁移过程不得调用大模型，也不得伪造学科来源。

## 四、学习资产与学习者模型

题目、掌握标准和课程误区保留自己的领域身份，同时增加正式知识引用和教学标准引用。课程误区即使未命中正式易错点也可以继续存在，但必须标记为课程局部内容。

学习者模型继续从正式事实确定性重算。每个课程目标携带正式知识 ID，模型按这些 ID 投影知识状态；一次阅读、一次错误或一次 AI 判断仍不能直接形成稳定掌握结论。

## 五、AI 老师

AI 上下文只加载当前节点相关的有限知识切片与教学标准切片。AI 先根据课程正文、题目和正式证据判断，再将结论标记为：

- 命中：标准库已有精确条目且当前证据支持。
- 部分命中：方向相符但颗粒度不足。
- 未命中：正式库无法解释当前问题。

部分命中和未命中可以形成待审核候选，但不能由一次对话直接晋升为正式标准。

## 六、学生端与治理端

学生端知识入口展示学科语义树、当前课程覆盖、相关能力、常见误区、提升方向和正文位置。课程目录继续留在学习页左栏，两者不重复。

正式知识维护和候选审核属于未来教师或管理工作台，不混入学生学习弹层。当前实现只提供只读正式学科包和课程覆盖视图。

## 七、新旧收束

- 删除 `knowledge_tree.py` 的课程五层真源实现。
- 删除未被当前前端使用的旧知识图谱路由。
- `learning_assets` 的正式资产改为 `subject_knowledge`、`teaching_standards` 和 `course_knowledge_map`。
- 旧 `knowledge_graph` 只作为迁移输入读取，不再作为 API 输出或活动资产。
- 前端组件和状态统一改名为知识库，不保留第二套知识树入口。
