# 设计：浏览器缓存只作离线兜底

## 1. 真源顺序

```text
服务端任务列表
  -> 本地活动任务是否已出现
  -> 缺失时查询服务端单任务
     -> 存在：补回列表并同步
     -> 404：清理失效本地状态
     -> 网络错误：保留最后成功状态
```

浏览器缓存只用于刷新和断网时保留现场，不能独立证明任务仍在运行。

## 2. 清理边界

只核对需要服务端支撑的活动或异常状态。正式完成任务不因列表分页而被清理。只有单任务接口明确返回 404 才清理；超时、断网、5xx 和其他未知错误均保留原状态。

## 3. 用户反馈

清理发生时显示一次本地化提示，任务中心与课程卡立即停止显示旧进度。该提示不建立新的失败状态或恢复动作。
