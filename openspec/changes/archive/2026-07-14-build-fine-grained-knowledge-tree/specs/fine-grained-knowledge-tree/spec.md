## ADDED Requirements

### Requirement: 正式知识结构必须是一棵五层细知识树

系统 MUST 将课程知识结构编译为课程、章节、小节、主题和知识点五类节点组成的树。每个节点 MUST 拥有稳定 ID、稳定编号、父级、完整路径、顺序、来源状态和不可变修订；树 MUST 恰有一个根，所有叶子 MUST 为知识点。

#### Scenario: 编译一门含两章的课程

- **WHEN** 正式课程包含两个章节及其学习小节
- **THEN** 系统 MUST 生成一个课程根及对应章节、小节、主题和知识点节点
- **AND** 每个小节 MUST 至少包含一个知识点叶子

#### Scenario: 重复编译相同课程

- **WHEN** 相同课程结构与知识内容被重复编译
- **THEN** 节点 ID、编号、路径和修订 MUST 保持一致

### Requirement: 新课程大纲必须声明可教学的细知识结构

新课程的每个学习小节 MUST 声明主题与细知识点。知识点 MUST 包含可区分名称、说明、可观察学习动作、别名和显式前置名称；缺失结构时系统 MAY 从旧 `key_points` 降级编译，但 MUST 标记为 `outline_only`。

#### Scenario: 模型返回完整知识结构

- **WHEN** 大纲模型为一个小节返回多个主题和知识点
- **THEN** 规范化器 MUST 保留合法主题、知识点说明、能力动作、别名和前置名称
- **AND** 正式知识树 MUST 标记这些叶子为已细化

#### Scenario: 旧模型只返回关键点

- **WHEN** 大纲只有 `key_points` 而没有 `knowledge_structure`
- **THEN** 系统 MUST 确定性建立“核心知识”主题和知识点叶子
- **AND** MUST 标记细化状态为 `outline_only`

### Requirement: 学习资产必须围绕同一知识点身份绑定

正文块、题目、掌握标准、通用误区、诊断、补救和证据 MUST 引用正式知识树中的知识点 ID，不得复制名称生成另一套概念身份。能力、误区、题目和个人掌握状态 MUST NOT 成为树节点。

#### Scenario: 编译一节的正式题目与误区

- **WHEN** 系统完成知识树并编译该节学习资产
- **THEN** 题目、掌握标准和误区中的知识点引用 MUST 指向该节正式叶子
- **AND** 引用不存在的知识点 MUST 使质量闸门失败

#### Scenario: 正文块明确提到一个知识点

- **WHEN** 正文块内容匹配知识点名称或别名
- **THEN** 块元数据与课程文档块 MUST 保存该知识点 ID
- **AND** 未精确匹配的块 MUST NOT 伪造精确知识点引用

### Requirement: 树结构与跨点语义关系必须分离

父子字段 MUST 作为唯一包含结构；跨知识点的前置、推导、对比、应用和一般关联 MUST 作为独立关系保存来源、状态、理由和修订。AI 推断关系 MUST 保持候选状态直至维护流程接受。

#### Scenario: 小节显式依赖前一小节

- **WHEN** 课程蓝图声明小节前置依赖
- **THEN** 系统 MUST 在两个小节的代表性知识点之间建立正式前置关系
- **AND** MUST NOT 用普通关系边重复整棵父子树

### Requirement: 旧课程和旧接口必须由正式知识树兼容投影

旧课程 MAY 从章节、小节和关键点确定性编译知识树。旧图谱接口 MAY 返回扁平概念格式，但 MUST 从正式知识树的知识点叶子实时投影，MUST NOT 保存或激活第二份概念真源。

#### Scenario: 旧客户端读取图谱

- **WHEN** 旧客户端请求只读知识图谱接口
- **THEN** 返回的概念 MUST 来自正式知识树叶子
- **AND** 关系端点 MUST 被映射为旧字段而不改变正式资产

### Requirement: 学生端必须以知识树承担导航与理解

学生端 MUST 通过学习页弹层展示可搜索、可折叠的课程知识树，并提供路径、说明、学习动作、误区、正文引用和跨点关系详情。学生端 MUST NOT 暴露知识树编辑、候选关系审批或课程顺序编排。

#### Scenario: 学生查看一个知识点

- **WHEN** 学生在树中选择一个知识点
- **THEN** 页面 MUST 展示完整路径及已有正式绑定
- **AND** 学生 MUST 能跳回对应课程正文

#### Scenario: 移动端打开知识树

- **WHEN** 390px 宽度设备打开知识树并进入详情
- **THEN** 页面 MUST 使用单栏树与可返回详情流
- **AND** 文本、操作和底部导航 MUST 不重叠或横向溢出

### Requirement: 知识树质量闸门必须阻止结构和引用错误

系统 MUST 检查唯一根、合法父级、无环、唯一 ID/编号、路径一致、叶子类型、小节覆盖、关系端点和所有学习资产引用。任一关键错误 MUST 生成具体质量问题并阻止资产激活。

#### Scenario: 知识点引用不存在

- **WHEN** 题目、误区或正文块引用知识树中不存在的 ID
- **THEN** 质量报告 MUST 包含 critical 引用问题
- **AND** 该资产修订 MUST NOT 激活
