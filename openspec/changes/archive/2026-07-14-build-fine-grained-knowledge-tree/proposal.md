# 提案：建立课程级细粒度知识树

## 背景

当前正式学习资产把每个学习单元的 `key_points` 编译成扁平概念列表，再用相邻顺序连线。它没有课程、章节、小节、主题和知识点层级，也没有稳定路径、块引用或能力/误区/题目绑定。前端因此只能把零散概念画成一张关系网，既不能承担课程导航，也不能成为学习事实、学习者模型与 AI 老师的共同语义坐标。

Code 平台已经验证了一条可复用的领域边界：统一知识树止于细知识点，能力、误区、题目和学习证据挂在知识点下，不混入树中。灵知需要继承这条结构原则，但不能复制编程学科的硬编码目录。

## 目标

1. 把正式 `knowledge_graph` 资产升级为唯一的 `knowledge_tree_v1` 结构真源。
2. 建立“课程 → 章节 → 小节 → 主题 → 知识点”的细粒度知识树，并为每个节点保存稳定编号、父级、路径、顺序、来源和修订。
3. 让课程正文块、题目、掌握标准、通用误区和资料证据引用同一批知识点 ID。
4. 为旧课程提供确定性只读编译，为新课程增加可生成细知识结构的正式契约。
5. 把学生端旧力导向图替换为轻量、可搜索、可折叠、可追溯到正文的知识树。

## 非目标

- 不把能力、误区、题目、笔记或个人掌握状态变成树节点。
- 不在本轮伪造知识点级掌握度；没有正式知识点证据时只展示课程结构与已有资产绑定。
- 不向学生开放知识树编辑、AI 候选关系审批或课程顺序编排。
- 不建立第二份知识图谱文件、第二套概念 ID 或第二个学习资产接口。

## 成功标准

- 任一正式课程恰有一个根节点，所有叶子均为知识点，每个学习小节至少含一个知识点。
- 新旧课程都可编译出稳定、无环、引用完整的知识树；相同输入重复编译得到相同节点 ID 和修订。
- 题目、误区、掌握标准与正文块引用的知识点全部存在于同一正式资产中。
- 旧 `/knowledge_graph` 接口仍可只读消费，但其扁平概念由知识树投影，不再独立存储。
- 学生可在弹层内搜索、展开、查看知识点详情并跳回对应课程正文；移动端无横向溢出。
