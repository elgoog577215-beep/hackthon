# 设计：一棵树承载课程语义，关联资产围绕叶子协同

## 1. 领域边界

正式学习资产继续使用 `assets.knowledge_graph` 这个既有槽位，避免建立第二真源；槽位中的对象升级为 `schema_version=knowledge_tree_v1`。

```text
课程根节点
└── 章节
    └── 小节
        └── 主题
            └── 细知识点
                ├── 正文块引用
                ├── 学习目标 / 掌握标准引用
                ├── 正式题目引用
                ├── 通用误区引用
                └── 资料证据引用
```

父子关系只表达包含结构，不重复保存为普通关系边。`prerequisite / derives / contrasts_with / applies_to / related` 只表达跨分支或跨知识点语义。

## 2. 正式数据契约

知识树对象包含：

- `schema_version`、`graph_id`、`root_node_id`、`revision_id`。
- `nodes`：稳定 `knowledge_id/code`、`parent_id`、`node_type`、`name/description`、`depth/sort_order`、`path_ids/path_names`、`aliases`、课程节点与正文块引用、目标/标准/题目/误区/证据引用、来源状态和细化状态。
- `relations`：稳定关系 ID、源/目标知识点、关系类型、来源、状态、理由与修订。
- `candidate_relations`：仍保留在正式资产中，但只供维护流程审阅，不进入学生学习界面。

节点类型固定为 `course / chapter / section / topic / knowledge_point`。树叶必须是 `knowledge_point`；主题不得在没有知识点时成为叶子。

## 3. 新旧课程编译

### 3.1 新课程

大纲生成契约在每个学习小节增加 `knowledge_structure`：每节包含 2-5 个主题，每个主题包含 2-5 个可单独解释、练习和诊断的知识点。每个知识点必须说明对象、成立条件或适用边界，并给出可观察学习动作、别名和显式前置名称。

规范化器在模型缺字段、字段不合法或旧 provider 不支持时，从 `key_points` 确定性生成一个“核心知识”主题，保证主链可用但标记 `detail_status=outline_only`，不得谎称已完成深度细化。

### 3.2 旧课程

旧课程不进行隐式 LLM 重写。编译器根据现有 L1/L2 课程节点和 `key_points` 建立五层树：课程、章节、小节来自正式结构；主题使用已有 `knowledge_structure`，缺失时使用小节下的“核心知识”；`key_points` 成为叶子。相同输入必须得到相同 ID、路径和修订。

## 4. 资产绑定

编译顺序为：

1. 先编译知识树并获得每个小节的叶子知识点。
2. 再编译题目、掌握标准、误区、诊断与补救资产，并写入正式知识点 ID。
3. 根据知识点名称、别名与正文块内容做确定性块级匹配；命中的知识点写入块元数据和课程文档 `concept_refs`。
4. 无法精确匹配时只保留小节级绑定，不把整节所有知识点伪装成某个具体段落的精确引用。

课程文档读取块级 `metadata.concept_refs`；只有旧块缺少块级引用时才使用节点级兼容引用。

## 5. 兼容与清理

- 正式学习资产不再持久化第二份 `concepts`。
- 旧 `/knowledge_graph` 路由从 `knowledge_point` 节点投影旧 `concepts` 和旧关系字段，保持只读兼容。
- 旧 AI 图谱生成器、运行时图谱注册表和力导向图组件在确认无生产引用后退出，避免新旧链继续并存。
- 候选关系接受/拒绝仍可由维护接口处理，但学生弹层不暴露编辑模式。

## 6. 质量闸门

结构闸门检查唯一根、合法父级、无环、唯一编号、深度和路径一致、叶子类型正确、每节有知识点。

引用闸门检查正文块、题目、掌握标准和误区中的知识点 ID 均存在，关系端点均为正式知识点。

语义闸门检查新课程知识点描述与学习动作不为空；旧课程的降级节点允许 `outline_only`，但必须显式标记。

必选知识树失败时，课程资产质量报告必须阻止当前资产激活。

## 7. 学生端体验

知识树继续从学习页底部工具坞以弹层打开，继承 3 月版浅色、轻边框、品牌色与紧凑层级：

- 顶部：课程名、搜索、知识点数量、关闭。
- 左侧：可折叠课程树，章节和小节承担导航，知识点显示轻量状态。
- 右侧：路径、知识点说明、学习动作、常见误区、关联正文和跨点关系。
- 点击正文引用可关闭弹层并回到原学习节点；不增加永久页面或顶部模式。
- 移动端使用单栏树，详情覆盖进入并提供返回，不缩成两列窄栏。
