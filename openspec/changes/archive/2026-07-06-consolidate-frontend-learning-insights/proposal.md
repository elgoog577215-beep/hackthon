# 收敛前端 AI 学习洞察体验

## Why

后端已经形成 `LearningEvent -> LearnerState -> TeachingDecision -> AI Learning Context` 的主链路，前端也已经出现结构化内容块、学习画像、学习统计、AI 助手和导师行动卡。但这些入口目前仍有职责重叠：统计面板会本地生成学习建议，画像面板展示动态状态，导师卡也给下一步行动，AI 助手里又嵌完整画像。

如果继续直接叠功能，用户会看到多个“看起来都很智能”的入口，却不知道该相信哪个。系统也会变成多个局部状态源互相竞争，削弱后端主链路的价值。

本轮目标是做一次优胜劣汰式合并：保留旧版统计和画像的好用展示，抛弃重复建议逻辑，把下一步行动统一交给 `TeachingDecision`，让前端形成清楚、轻量、可持续的学习体验。

## What Changes

- 新增前端统一学习洞察组合层，把 `tutorStore.learningState`、`TeachingDecision`、旧统计、画像和错题事实整理成一套 UI 可消费对象。
- `TutorActionCard` 成为当前学习页唯一主动建议入口，建议来源优先来自 `TeachingDecision`。
- `LearnerProfile` 降级为画像和判断依据解释层，不再像第二个看板一样承载全部状态。
- `LearningStats` 改成学习轨迹面板，保留时长、连续学习、完成率、测验统计、知识图谱和报告，移除本地规则式“下一步建议”的主导权。
- `SideAIPanel` 保留对话能力，只展示轻量学习状态摘要，不再嵌完整画像面板。

## Non-Goals

- 不新建大型驾驶舱页面。
- 不改后端 AI Learning OS 主链路。
- 不删除旧统计、画像、AI 助手或测验功能。
- 不引入新 UI 框架或重型状态管理。
- 不大改课程阅读主布局。

## Acceptance Criteria

- 前端只有一个主动“下一步学习建议”入口：导师行动卡。
- 学习统计面板不再用本地规则生成另一套学习建议，而是展示学习轨迹和来自统一洞察的简洁提示。
- 学习画像面板能清楚区分长期画像、动态证据和系统推断依据。
- AI 助手面板不再嵌完整画像组件，只展示轻量状态摘要并保留聊天输入。
- 旧入口保持可用，用户仍能查看统计、画像、错题、图谱和 AI 助手。
- 前端构建、测试、OpenSpec 校验和 `git diff --check` 通过。
