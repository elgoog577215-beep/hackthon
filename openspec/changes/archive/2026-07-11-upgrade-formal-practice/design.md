# 设计

## 1. 真源边界

- `QuestionRevision` 是正式题目真源，属于课程学习资产。
- `PracticeAttempt` 是一次真实作答的当前对象；提交后答案不可覆盖，重试创建新 Attempt。
- `LearningEvent` 保存开始、提示、提交、评分、放弃、失效和旧数据导入事实。
- `ObjectivePracticeResult` 与掌握状态都是投影，不建立第二份可修改真源。

## 2. Attempt 生命周期

服务端状态只允许 `in_progress / submitted / grading / graded / abandoned / invalidated`。活动草稿按期望修订号更新；提交携带幂等请求 ID。课程或题目发生不兼容变化时保留草稿并标记 invalidated，不允许把旧答案提交到新标准。

草稿即时写本地缓存并延迟同步服务端。服务端是跨设备真源；冲突不得静默覆盖。离线可以继续编辑但不能伪装正式提交成功。

## 3. 提示、评分与证据

提示分为方向、关键步骤和解题框架三级；完整解析单独记录。AI 在正式练习中的帮助映射到相同支持等级。评分结果保存量规分项、评分方法、置信度和 `independent / lightly_supported / supported / scaffolded / invalid` 证据强度。

确定性题目由程序评分。开放题使用严格量规和结构化 AI 辅助评分；失败、低置信或通过线附近结果进入待评阅，不得退回关键词伪精确评分。

## 4. 掌握投影

理解检查不直接证明掌握；目标练习形成能力证据；掌握检查和综合检测可以验证掌握。查看完整解析或使用三级提示后在同题重做只算纠正证据，必须用未泄露答案的当前版本题目再次验证。

投影保留 `not_checked / evidence_insufficient / partial / mastered / needs_review`。只有当前目标、题目和标准修订的有效证据参与当前状态，历史证据永久保留但不得冒充当前掌握。

## 5. 前端工作区

所有入口打开同一主内容练习工作区，一次只呈现一道题。页面显示目标、层级、进度、草稿状态、结构化作答、提示、问老师、提交、反馈和唯一下一步。练习历史与待巩固归入练习工作区，错题不再属于学习记录面板。

## 6. 迁移与删除

旧 `quiz_wrong_answers`、`quiz_history` 和单数 `learning_record.py` 数据幂等导入为低置信历史事件，不创建正式 Attempt 或掌握证据。只有成功迁移的浏览器项目被删除，未匹配其他课程的项目继续保留等待迁移。

新消费者切换完成后删除旧测验弹窗、错题闯关、类似题入口、本地错题 Store、旧全局草稿、旧测验分析服务和旧 Prompt 注册。AI 问答从误导性的 quiz 路由归位到 assistant 路由，保持外部 URL 兼容。
