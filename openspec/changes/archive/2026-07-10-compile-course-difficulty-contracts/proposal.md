# 课程难度契约化

## Why

当前课程生成虽然保存 `beginner/intermediate/advanced`，但它们还只是字符串标签。蓝图 prompt 只告诉模型“当前难度”，小节的 `complexity` 由模型自由填写且没有生产约束；前端又将难度映射成公式密度、子节数量和通用文案。这会导致伪难度：入门版只是更短，高阶版只是更长、术语更多，而学习任务、支架和验收能力没有真正变化。

## What Changes

- 将课程难度从文案标签升级为可持久化、可编译、可验证的能力契约。
- 使八种教学模式使用同一组难度维度，再由学科适配器翻译为实际学习任务。
- 让课程蓝图、小节正文、练习与质量闸门消费同一份难度契约。
- 保留 API 的三个稳定值，删除“公式密度 = 难度”、“子节数 = 难度”和节点重写默认高级等旧逻辑。

## 范围

- 新增后端难度编译器、课程曲线、节点契约、旧课程适配和难度质量报告。
- 升级 `CourseService`、课程蓝图、正文 prompt、定向修复、节点重写和最终质量报告。
- 升级前端难度选择文案与局部再生成参数。
- 增加八模式三等级的确定性契约基准和主链测试。

## 不在本次范围

- 不新增第二个课程生成服务或任务系统。
- 不根据学习事件在用户不知情时自动降低或提高已设定的目标难度。
- 不在每个学科下维护三套独立 prompt。
- 不依赖真实大模型调用才能完成契约编译与基础验证。

## 兼容性

- 对外难度值继续使用 `beginner/intermediate/advanced`。
- 旧课程没有结构化契约时，读取适配器根据已保存的难度和教学画像恢复最小契约，不重启旧生成链。
- 当用户未提供可判定的当前基础时，入口要求状态为 `unknown`，不伪造学习者水平。
