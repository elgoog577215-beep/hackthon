# 统一 AI Learning OS 后端智能链路

## 背景

当前后端已经完成两块基础能力：`LearningEvent` 负责记录学习证据，`LearnerState` 负责把事件聚合成可查询状态。但 AI 助手、课程内容生成、导师建议和旧学习者上下文仍然分散消费不同信号，容易继续长成多条并行电路。

这次变更的目标不是再加一个接口，而是把事件、状态、教学决策和 AI 上下文编排收束成同一条轻量主链路：

```text
LearningEvent → LearnerState → TeachingDecision → AI Learning Context → AI 助手/课件生成/导师建议
```

## 范围

- 新增轻量教学决策层，把 `LearnerState` 转换成结构化教学意图。
- 新增统一 AI Learning Context 编排层，组合课程、节点、学习者上下文、学习者状态和教学决策。
- 将 AI 助手 prompt 接入统一上下文和教学决策摘要。
- 将课程内容生成、重写、扩展、摘要链路轻量接入统一上下文。
- 将导师主动建议优先读取新状态/决策，旧导师记忆继续作为兼容补充。
- 更新架构文档，说明新主链路和职责边界。
- 补充测试，验证决策、上下文、助手和课程链路兼容。

## 非目标

- 不改前端 UI。
- 不引入重型数据库、向量库或异步状态同步系统。
- 不把 `LearnerState` 替换为大段 prompt。
- 不让教学决策层直接调用大模型。
- 不删除仍有生产引用的旧接口。

## 验收标准

- `TeachingDecision` 能从 `LearnerState` 生成可解释、结构化的教学动作建议。
- `AI Learning Context` 能输出机器可读结构和 prompt 摘要。
- AI 助手 prompt 同时包含学习者状态摘要和教学决策摘要。
- 课程内容生成/重写/扩展/摘要至少通过统一上下文读取状态和决策摘要。
- 导师建议可从状态/决策生成复习、练习、简化解释等建议，同时保持旧接口返回结构兼容。
- 旧 `LearnerContext` 继续可用，但成为统一上下文的一部分。
- 相关 pytest、Python 编译检查、OpenSpec 校验和 `git diff --check` 通过。
