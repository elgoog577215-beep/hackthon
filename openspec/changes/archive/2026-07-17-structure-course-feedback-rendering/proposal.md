# 变更：结构化课程检查与反馈并修复正文阅读排版

## Why

当前课程已经能按二级标题生成教学块，但“检查与反馈”块内部仍是一段自由 Markdown。多个任务、答案方向、推导、评价标准和常见错误会被压进同一个 `rich_text`，数学表达还经常被反引号写成代码。前端只识别外层 `feedback` 角色，无法逐项呈现；全局样式重置后，列表序号、缩进和段落节奏也没有在正文渲染器中恢复，最终形成答案墙。

这不是单纯 CSS 问题。生成契约、正式课程块、质量门和学习页必须共同建立一个可执行的反馈呈现契约。

## What Changes

1. 明确静态课程块只保存参考结论、核对标准和常见错误；基于学生作答的个性化反馈继续归学习痕迹与 AI 老师。
2. `feedback` 课程块编译为 `review_checkpoint`，并保存版本化 `feedback_structure`，将任务级内容拆成稳定小节。
3. 课程生成要求反馈块使用任务级三级标题、真实 Markdown 列表和 LaTeX 数学分隔符，禁止用反引号伪装公式。
4. 质量门拒绝长反馈块缺少任务层级、数学表达大规模退化成行内代码等结构缺陷。
5. 学习页使用专用反馈组件逐项折叠参考内容，并恢复 Markdown 标题、列表、段落、引用、表格和行内代码的阅读排版。
6. 对旧课程提供只读兼容解析，不改写既有正式课程内容或学习事实。

## Out of Scope

- 不自动重写既有课程答案或修改其学科结论。
- 不把静态参考答案冒充学生个性化反馈。
- 不新建第二套课程正文、练习或 AI 老师入口。
- 不改变正式练习的作答、评分和学习证据协议。

## Success Criteria

- 截图中的旧“检查与反馈”内容能按任务拆成可逐项展开的区域，不再首屏显示整块答案墙。
- 列表恢复序号与缩进，公式优先走 KaTeX，正文段落具有稳定阅读节奏。
- 新生成课程的长反馈块必须有任务级内部结构，数学内容不得批量使用反引号表达。
- 中文与英文模式、桌面与窄屏均无原始翻译 key、遮挡和横向正文溢出。
- 后端契约测试、前端组件测试、生产构建和真实页面截图验证通过。
