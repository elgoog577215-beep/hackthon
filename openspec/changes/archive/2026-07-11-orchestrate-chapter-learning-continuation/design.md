# 设计

## 1. 真源与投影边界

`LearningContinuation` 是只读协调投影，不拥有学习事实。它读取：

- 课程当前版本、L1 章节、L2 目标与 `prerequisite_node_ids`。
- `LearningProgress` 的阅读和掌握证据。
- `LearningSnapshot` 的当前位置与当前任务。
- `PracticeAttempt` 的未完成正式练习。
- `DiagnosticCase` 与 `RemediationSession` 的活动状态。
- `LearningRecord` 的明确阻塞问题和到期复习。

章节当前真相不得保存为一个可修改的 `chapter_status`。章节结果与下一步每次由当前输入修订重新计算。

## 2. 章节、目标和推进契约

正常课程以 L1 节点为章节、其 L2 子节点为学习目标。只有 L1 内容的历史课程按每个可学习 L1 节点投影一个兼容章节和目标，不写回课程资产。

课程可以在学习资产中提供 `ChapterProgressionContract`：必需目标、是否要求掌握验证、前置策略和完成策略。缺少契约时使用兼容默认值：阅读完成即可推进、掌握验证不阻断、前置依赖只提醒。

## 3. 三维状态

- 学习进度：`not_started / in_progress / covered`。
- 掌握证据：`not_checked / evidence_insufficient / partial / verified / needs_attention`。
- 任务连续性：`none / resumable / stale / blocked`。

前端由投影显示首次进入、继续学习、恢复任务、处理版本变化、完成验证和章节收束，不另建章节首页。

## 4. 风险规则

风险固定为 `notice / action_recommended / action_required`。只有显式必需前置依赖和当前可靠证据可以阻断；证据缺失只能标记尚未验证。停留时间、一次答错、旧错题数量、低置信迁移和 AI 推断不得升级风险。

非阻断风险可以延后，延后写为 `entry_risk_deferred` LearningEvent 并带冷却时间。阻断风险不可延后。

## 5. 唯一下一步

`NextLearningAction` 至少包含动作类型、目标、原因码、证据引用、是否阻断、是否需要确认和可用性。优先级固定为：

1. 课程版本失效确认。
2. 活动诊断、补救或独立复验。
3. 未提交正式练习。
4. 用户明确标记的阻塞问题。
5. 可靠且必需的前置缺口。
6. 恢复阅读位置。
7. 开始或继续当前目标。
8. 掌握检查。
9. 遗留问题与到期复习。
10. 查看章节结果、进入下一章或结束课程。

待评阅不得阻断学习。并列任务依次按当前章节、快照指向、有草稿和更早创建时间裁决；仍无法裁决时输出任务冲突确认动作。

## 6. 前端职责

`LearningContinuityBar` 位于工作区导航下方，显示章节与目标、进入模式、最强原因、一个主要动作和最多两个次要提醒。它只负责跨模块导航或明确完成阅读；进入正式练习、诊断、补救等工作区后，提交动作仍由对应工作区负责。

`TutorActionCard` 只解释统一动作并提供问 AI，不执行练习或复习。`SmartBar` 只保留平级工具入口。章节结果使用无嵌套卡片的目标行展示，不显示总分冒充掌握。

## 7. 刷新、并发和版本

进入课程、切换章节或目标、学习进度变化、Attempt 生命周期变化、诊断生命周期变化、相关学习记录变化、版本变化和网络恢复时刷新投影。滚动、草稿每次输入和计时器不得刷新。

`projection_revision_id` 由所有输入的稳定修订计算，不包含生成时间。版本变化保留历史作答、诊断和记录，先输出 `confirm_version_change`，不得静默复制、删除或重映射。

## 8. AI 边界

AI 可以读取 `NextLearningAction`、风险和章节结果并解释原因，但不得创建动作、升级风险、改变优先级或写入章节完成状态。
