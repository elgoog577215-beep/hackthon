# 设计：课程降级学习路径

## 1. 单一判断源

新增纯函数课程学习可用性投影，不写数据库，也不生成缺失资产。所有消费方只读取同一份投影：

- `standard`：现代课程且未声明纯阅读，完整学习链要求正式练习。
- `reading_only`：`learning_asset_plan.reading_only_degraded=true`，阅读和章节推进可用，练习、掌握证据和自动补救按设计关闭。
- `compatibility`：课程缺少现代生成契约的整体结构，旧正文可继续阅读，但不声称具备正式练习或掌握证明。

兼容识别必须基于生成时代结构，而不是题目数量。现代课程即使缺题也仍是 `standard`，其练习能力为 `blocked`。

## 2. 能力状态

契约分别表达阅读、练习、掌握证据和自动补救。练习接口在当前范围无题时返回以下原因之一：

- `declared_reading_only`：课程按设计只支持阅读。
- `legacy_reading_compatible`：旧课程只保证正文学习。
- `required_practice_missing`：标准课程缺少必需正式题目，属于资产故障。
- `no_questions_in_scope`：课程整体有题，但当前节点或范围没有题。

不得把上述状态压缩成一个通用空列表提示。

## 3. 连续性边界

纯阅读和旧课兼容沿用 `reading_covered` 推进：阅读完成不等于掌握，章节结果保持 `covered_unverified`，但可进入下一章。

标准课程在必需练习缺失时可以继续完成现有正文阅读；当下一步需要正式掌握检查时，LearningContinuation 必须返回不可用的 `repair_course_assets` 主行动，说明这是课程资产问题，不得伪造掌握，也不得打开空练习。

## 4. AI 模型不可用

AI 模型失败不得改写课程使用模式、学习进度、快照、正式记录或连续性动作。AI 老师只返回安全的 `model_unavailable` 结果，不泄露 provider 细节；用户仍可执行确定性学习动作。

## 5. 验收边界

后端使用固定课程夹具验证三类模式和标准缺题阻断；前端验证四类无题原因的明确文案；现有旧课程通过公开 API 验证可开始阅读。AI 不可用使用受控 provider 失败验证，避免修改全局生产配置。
