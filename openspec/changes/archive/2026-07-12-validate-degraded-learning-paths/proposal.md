# 变更：验收课程降级学习路径

## Why

标准课程主学习链已经通过真实 API 验收，但课程库仍同时存在旧课程、显式纯阅读课程和现代标准课程。当前正式练习接口只返回空题目列表，前端无法区分“设计上不生成题目”“旧课程只保证阅读”与“标准课程资产异常”，容易把正常降级误报为故障，也可能把真实生成故障静默放宽。

第 8.5C 步需要建立一份只读课程使用契约，让 LearningRuntime、正式练习和前端使用同一判断，并验证模型不可用时不会破坏确定性的课程学习链。

## What Changes

- 增加只读课程学习可用性契约，统一表达 `standard / reading_only / compatibility` 三种模式及各能力可用性。
- 兼容模式只根据旧生成时代的明确结构特征识别；不得因为现代课程缺题而自动进入兼容模式。
- 正式练习接口返回无题原因和可用性；前端分别展示纯阅读、旧课兼容、当前范围无题和标准资产异常。
- LearningRuntime 暴露同一契约；标准课程完成阅读后若缺少必需练习，连续性动作必须明确阻断，不得打开空练习。
- 验证 AI 模型不可用时只降级 AI 回答，阅读、进度、记录和确定性下一步仍可继续。

## 影响范围

- 后端：课程使用契约、LearningRuntime、LearningContinuation、正式练习接口。
- 前端：练习空状态、章节掌握状态和运行时类型。
- 验收：旧课程、显式纯阅读课程、现代标准课程资产异常和 AI 模型不可用回归。
