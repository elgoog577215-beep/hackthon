# 设计：标准课程主学习链验收

## 1. 验收对象

标准课程固定为 `4dcfe257-0955-49bb-ade4-dc6ed915bbfb`，当前课程版本必须通过 strict preflight 且保持 ready。主验收用户每次使用 `acceptance-main-user-<run>` 前缀的全新身份，设备 A 与设备 B 分别为 `acceptance-main-device-a` 和 `acceptance-main-device-b`。

## 2. 执行边界

验收器只能调用生产公开 API。它可以读取课程与运行时、创建验收用户自己的快照、记录、Attempt、诊断工作流、AI 会话和动作回执；不得直接编辑课程、学习事件、Attempt、诊断或快照持久文件。

每个阶段保存结构化证据摘要，包括请求场景、关键对象 ID、前后状态、运行时修订与断言结果。敏感凭据、完整模型 prompt、完整答案和个人数据不得进入证据文件。

## 3. 主链顺序

1. `ENT`：确认首次进入指向当前版本、第一章和第一个目标，并只有一个 primary action。
2. `PRG`：显式 start 与 complete_reading，确认阅读完成不等于掌握。
3. `SNP`：设备 A 保存语义锚点与活动任务，设备 B 读取同一服务端快照；并发旧修订必须返回 409。
4. `REC`：创建四类学习记录，验证更新、归档和并发冲突，确认 LearningRuntime 汇总来自正式记录。
5. `PRA`：创建正式 Attempt、保存草稿、跨设备恢复、记录帮助、提交与重复提交，确认历史不可覆盖且支持等级进入证据。
6. `DIA`：制造两次高置信低支持失败，确认单次失败不建案、第二次失败只建立一个诊断案例，并通过辨别任务验证假设。
7. `REM`：完成最小补救，引导题不得结案；使用未见、零帮助独立复验通过后才 resolved。
8. `CON`：确认每个阶段只有一个主行动，章节结果能够解释目标证据与遗留项。
9. `AIT`：从正文、练习和连续性上下文进入同一协议，验证来源绑定、答案隔离、模型错误安全降级和可确认动作回执。

## 4. 停止与修复

发现 P0 时立即停止；发现 P1 时按 `DOMAIN / RUNTIME / AI / FE / ASSET` 的唯一主归属修复，并从最近稳定门禁重跑。不得在验收脚本中伪造课程资产、评分结果、诊断结论或掌握状态。

## 5. 收束条件

九段主链全部通过，标准课程 strict preflight 仍为 ready，相关回归测试与全量测试通过，证据已回写文档，且 OpenSpec 严格校验通过后，8.4 才能完成并归档。
