# 变更：验收标准课程主学习链

## Why

课程使用前七步已经分别实现，8.3 也已经生成并发布一门具备六项课程使用契约的真实标准课程，但局部测试与静态预检仍不能证明这些能力能在同一用户、同一课程版本和同一条真实学习过程中连续工作。

第 8.4 步必须使用隔离验收身份，仅通过生产公开 API，依次验证首次进入、阅读进度、跨设备快照、学习记录、正式练习、错因诊断、最小补救、独立复验、章节连续性和 AI 老师。任何需要直接修改持久文件才能继续的情况都视为主链阻断。

## What Changes

- 建立可重复执行的标准课程主学习链验收脚本，固定课程、用户、设备和证据格式。
- 逐段验证 `ENT -> PRG -> SNP -> REC -> PRA -> DIA -> REM -> CON -> AIT`。
- 验证每次领域写操作后 LearningRuntime 与唯一主行动同步更新。
- 对真实链路发现的 P0/P1 阻断按主归属修复，并增加回归测试。
- 将真实任务 ID、对象 ID、状态迁移与最终结果回写全链路验收矩阵和产品蓝图。

## 影响范围

- 后端：学习快照、学习进度、学习记录、正式练习、诊断补救、LearningContinuation、LearningRuntime 与 AI 老师协议。
- 验收工具：只调用公开 HTTP API，不新增业务状态或验收数据库。
- 数据隔离：每次使用 `acceptance-main-user-<run>` 前缀的全新用户与两台固定设备，不读取或覆盖默认用户数据。
