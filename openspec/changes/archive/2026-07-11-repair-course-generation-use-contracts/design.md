# 设计：课程生成到课程使用的发布契约

## 1. 生成收束顺序

标准生成固定为：正文定稿并持久化内容块 -> 编译学习资产与章节推进契约 -> 执行资产质量门 -> 执行课程总质量门 -> 保存不可变资产包修订 -> 创建或提升课程版本 -> 激活资产包 -> 保存当前课程。任一质量门失败时不得创建当前版本或激活资产包。

## 2. 内容块

`_save_generated_node_content` 使用现有 `set_node_content_blocks` 将最终正文转换为稳定块，并保存回节点。WebSocket completion/finalized 事件从保存后的节点读取内容块，不能重新解析或发送空数组。草稿阶段不创建正式内容块。

## 3. 章节推进契约

编译器按 L1 章节与直属 L2 目标生成 `chapter_progression_contracts`：

- `chapter_id` 使用真实 L1 `node_id`，与 LearningContinuation 完全一致。
- `required_objective_ids` 使用当前目标修订绑定的稳定目标 ID。
- 系统化、应试和个性化补救课程默认要求掌握；资料整理课程默认只要求阅读覆盖。
- 有显式跨节点前置关系的章节使用 required，否则使用 advisory。
- 每项契约有稳定 `asset_id` 与 `revision_id`。

## 4. 质量与发布

资产质量门检查所有目标节点存在持久内容块和块修订，所有章节存在有效推进契约，契约只引用当前目标。资产仓库保存不可变修订时 `activate=False`；初始版本创建或候选版本提升成功后才激活。

## 5. 验证边界

受控服务测试可以证明生成收束、存储和预检协议，但不能替代真实模型内容质量。真实标准样本必须通过正式生成 API 发起；若外部凭据失败，应保留任务错误和安全降级证据，不得手工构造 ready 课程。
