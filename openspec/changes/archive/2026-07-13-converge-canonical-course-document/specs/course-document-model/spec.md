## ADDED Requirements

### Requirement: 已迁移课程必须只有一个持久课程正文真源

系统 MUST 将已迁移课程的章节与有序内容块持久化在同一份 `CourseDocument` 中，并使用课程修订、块修订、内容指纹和稳定块标识保护修改。已迁移课程 MUST NOT 同时持久化第二份旧 `nodes` 正文副本。

#### Scenario: 读取已迁移的标准课程

- **WHEN** 客户端读取已经迁移的标准课程文档
- **THEN** 后端 MUST 返回持久化的 `CourseDocument`
- **AND** 章节与内容块顺序 MUST 与迁移前一致
- **AND** 课程文件 MUST NOT 依赖旧 `nodes` 作为正文真源

#### Scenario: 修改一个正式课程块

- **WHEN** 领域命令携带当前课程修订修改指定块
- **THEN** 系统 MUST 只更新目标课程文档
- **AND** 系统 MUST 更新相关修订并保存幂等操作回执
- **AND** 修订不匹配时 MUST 拒绝静默覆盖

### Requirement: 旧接口只能兼容读取，不能覆盖正式课程文档

系统 MUST 为旧读取调用从 `CourseDocument` 生成内存兼容投影，以维持尚未迁移的前端和领域服务；该投影 MUST NOT 成为独立持久状态。旧节点保存、旧自动生成任务及其他兼容写入 MUST 拒绝覆盖已迁移课程。

#### Scenario: 旧服务读取已迁移课程

- **WHEN** 旧服务通过课程读取接口请求已迁移课程
- **THEN** 仓库 MUST 从正式课程文档生成兼容节点投影
- **AND** 投影中的块标识与修订 MUST 保持稳定
- **AND** 系统 MUST NOT 将该投影另存为第二份正文

#### Scenario: 旧写入尝试覆盖已迁移课程

- **WHEN** 旧节点接口或旧生成任务尝试保存已迁移课程
- **THEN** 系统 MUST 明确拒绝该写入
- **AND** 原课程文档 MUST 保持不变

### Requirement: 迁移必须显式、可预览且可校验

历史课程迁移 MUST 先产生确定性的预览与来源校验值，只有显式提交且校验值匹配时才能替换正文真源。未迁移课程的正式文档接口 MAY 返回只读兼容文档，但 MUST NOT 因一次读取自动落盘迁移。

#### Scenario: 预览历史课程迁移

- **WHEN** 操作者请求历史课程迁移预览
- **THEN** 系统 MUST 返回来源校验值、章节数、块数和迁移结果摘要
- **AND** 原课程文件 MUST 保持不变

#### Scenario: 使用过期校验值提交迁移

- **WHEN** 来源课程在预览后发生变化且提交仍携带旧校验值
- **THEN** 系统 MUST 拒绝迁移
- **AND** 系统 MUST NOT 删除旧正文或写入部分课程文档

### Requirement: 首次课程生成必须使用隔离工作区

系统 MUST 在创建首次课程生成任务时建立统一课程文档外壳，并将蓝图、节点草稿、流式正文和质量结果保存到按任务隔离的工作区。正式课程文件 MUST NOT 持久化生成中的旧 `nodes` 正文。

#### Scenario: 新课程进入生成队列

- **WHEN** 用户提交一次新的课程生成请求
- **THEN** 系统 MUST 返回唯一 `job_id` 与 `course_id`
- **AND** 课程库 MUST 能读取使用同一 `course_id` 的统一课程文档外壳
- **AND** 外壳 MUST NOT 包含持久化旧 `nodes`
- **AND** 生成任务 MUST 在隔离工作区保存后续草稿

#### Scenario: 暂停并恢复生成任务

- **WHEN** 用户暂停任务或服务重启后恢复同一个任务
- **THEN** 系统 MUST 使用原 `job_id` 读取同一隔离工作区
- **AND** 已完成正文与未完成草稿 MUST 可继续使用
- **AND** 系统 MUST NOT 将工作区草稿提前复制到正式课程文档

#### Scenario: 审阅生成前蓝图

- **WHEN** 初始生成任务进入蓝图等待确认状态
- **THEN** 蓝图读取和修改 MUST 针对该任务的隔离工作区
- **AND** 正式课程文档 MUST 保持空外壳状态

### Requirement: 质量通过后必须原子发布正式课程文档

系统 MUST 只在课程质量、学习资产质量和节点完成状态全部通过后，将生成工作区编译为正式 `CourseDocument / CourseBlock` 并通过课程文档仓库一次性发布。发布 MUST 幂等，且 MUST NOT 创建旧节点正文副本。

#### Scenario: 首次生成质量全部通过

- **WHEN** 课程正文和正式学习资产均通过质量门
- **THEN** 系统 MUST 将章节、内容块、教学角色、目标引用和证据引用编译到正式课程文档
- **AND** 系统 MUST 在一次课程保存中更新正式文档、生成元数据、文档修订和操作回执
- **AND** 生成结果 MUST NOT 持久化旧 `nodes`
- **AND** 学习页 MUST 能通过正式课程文档接口读取刚发布的课程

#### Scenario: 发布命令重复执行

- **WHEN** 服务在发布后、任务完成状态落盘前重启并再次执行同一发布命令
- **THEN** 系统 MUST 返回原幂等回执
- **AND** 系统 MUST NOT 重复创建课程块、操作记录或第二份课程正文

#### Scenario: 课程或资产质量未通过

- **WHEN** 生成结果存在失败节点，或课程质量、学习资产质量任一未通过
- **THEN** 系统 MUST 保留工作区供诊断和重试
- **AND** 系统 MAY 更新课程外壳的失败状态与质量摘要
- **AND** 系统 MUST NOT 将任何生成中正文发布到正式课程文档
