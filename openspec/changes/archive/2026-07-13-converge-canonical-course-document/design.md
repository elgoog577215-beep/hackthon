# 设计：纵向接管而不是长期双写

## 1. 唯一真源

正式课程文件保留课程级生成、资料和学习资产元数据，正文只保存一个 `course_document`。旧 `nodes` 在迁移后不再持久化，只由兼容适配器按需生成。

## 2. 读取与写入

```text
旧课程读取 -> 内存投影 -> CourseDocument -> 前端课程流
已迁移课程 -> CourseDocumentRepository -> CourseDocument -> 前端课程流
正式修改 -> CourseCommandService -> CourseDocumentRepository -> 原子保存
```

旧保存入口遇到已接管课程必须返回冲突，不能重建并覆盖旧节点。首次课程生成通过下述隔离工作区接入正式仓库，不再调用旧课程保存器持续覆盖当前课程。

## 3. 修订与历史

- 文档修订用于防止整门课程并发覆盖。
- 块修订用于锚点、正式引用和局部修改冲突。
- 删除使用墓碑状态，不立即抹去历史块身份。
- 命令使用稳定命令 ID；重复提交返回原回执。
- 用户不可见的旧课程版本暂保留为历史证据映射，不恢复版本产品页面。

## 4. 第一条纵向链

标准课程先经过只读预览和校验，再受控迁移。前端课程 Store 从课程文档接口读取，临时投影为现有目录组件需要的节点视图；正文块本身直接来自课程文档，不重新以 Markdown 为真源。

## 5. 首次生成的隔离与发布

```text
创建唯一 GenerationJob
-> 建立空的 CourseDocument 外壳，供课程库显示课程身份
-> 建立按 job_id 隔离的生成工作区
-> 资料、蓝图、节点草稿、正文和质量结果只写工作区
-> 编译正式学习资产但暂不激活
-> 课程质量与资产质量同时通过
-> 将工作区编译为 CourseDocument / CourseBlock
-> 激活已通过质量门的学习资产
-> 由 CourseDocumentRepository 原子发布并记录幂等回执
-> 学习页读取正式课程文档
```

生成工作区允许继续使用节点形状承载模型编排、依赖调度和流式草稿，但它只是任务内部中间表示，不是课程真源，也不能被普通课程读取接口暴露。暂停、服务重启和重试必须重新读取同一个工作区。

课程文档外壳只包含课程身份、生成任务引用、生成状态和空的正式文档。蓝图审阅接口在存在等待确认的初始任务时读取工作区；确认后的修改仍写回该工作区。质量失败只更新外壳的失败状态与质量摘要，不复制工作区正文。

发布命令使用稳定 `job_id` 作为幂等边界，校验当前外壳仍属于该任务，再在一次课程保存中写入正式文档、课程元数据、文档修订和操作回执，并确保旧 `nodes` 不进入结果。学习资产先激活、课程后发布：如果课程发布失败，最多留下没有被当前课程引用的资产修订，不会产生引用未激活资产的正式课程。为兼容现有学习事实，`current_course_version_id` 暂时投影为当前课程文档修订；它不创建用户可见版本或第二份课程快照。

首次生成发布之外的旧候选版本和局部再生成路径暂不在本轮改写，但不得覆盖已经统一的课程文档。
