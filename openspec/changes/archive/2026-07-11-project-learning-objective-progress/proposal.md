# 变更：建立学习目标与进度投影

## 为什么

课程已经生成学习目标、正式题目和掌握标准，但课程使用仍把“打开有正文的节点”写成本地完成，并在统计页将完成节点显示成已掌握。这会混淆阅读事实、学习进度和掌握证据，也无法按用户、课程版本和目标修订稳定恢复。

## 改什么

- 为每个二级学习节点建立稳定 `LearningObjective` 与不可变修订，并绑定正文块、正式题目、掌握标准、误区和回退入口。
- 基于 `LearningEvent` 新增用户级学习进度投影，严格分离阅读进度与掌握状态。
- 打开节点只记录幂等的开始事件；用户明确确认后才能进入已学完；正式练习与掌握标准决定系统验证。
- 将旧 `completedNodes/is_read` 降级迁移为历史接触证据，不得直接形成已学完或已掌握。
- 前端以服务端投影显示当前目标、学习状态、掌握状态和唯一下一步操作，统计页退出本地完成数组真源。

## 影响

- 后端：学习目标投影、学习事件字段、进度 API、学习资产与旧统计兼容接口。
- 前端：学习进度 store、阅读状态条、课程工作区、学习统计和旧完成状态迁移。
- 规格：新增 `learning-progress-state` 能力规格，并补充正式资产与目标绑定要求。
