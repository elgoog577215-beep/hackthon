# 设计：正文预览态与专注作答态共用一份正式任务

## 1. 状态与真源

练习预览块只读取 `LearningAssets.questions` 中属于当前节点的正式题目，展示题量和首题 `prompt` 摘要。它不创建草稿、不推断完成度，也不保存第二份任务状态。

用户打开后继续由既有 `PracticeWorkspace` 加载正式题目、当前 `PracticeAttempt`、草稿、提示、评阅和历史。预览块与弹窗只是同一任务在“正文扫读”和“专注操作”两种表面上的投影。

## 2. 展开与返回

正文预览块触发时，`LearningView` 记录：

- 当前正文滚动位置；
- 触发元素；
- 预览块在视口中的矩形。

通用 `MorphingDialog` 在挂载后测量居中弹窗矩形，以 FLIP 思路从来源矩形过渡到目标矩形；任务正文稍后淡入，避免复杂表单被明显拉伸。关闭时反向过渡，待动画结束后才卸载弹窗，再恢复滚动与焦点。

若入口不是当前可见的正文预览块，不记录来源矩形，弹窗使用轻微缩放与淡入。系统不得把 Dock 或恢复入口伪装成正文块来源。

## 3. 弹窗规格

- 桌面：宽度不超过 `1040px`，与视口四周至少保留 `32px`；高度不超过 `84vh`，圆角和阴影使用学习现场现有 token。
- 移动：小于 `768px` 时占满可用视口，不保留装饰圆角；内部工作区独立滚动。
- 背景：低对比遮罩和轻微模糊，仍能感知正文来源，但不与任务争夺注意力。
- 关闭：标题栏按钮、Escape 和遮罩均可关闭；弹窗内部 Tab 焦点循环。
- 动效：默认约 340ms；`prefers-reduced-motion: reduce` 下取消位移缩放，仅保留即时状态切换。

## 4. 视觉边界

预览块使用正文同宽、普通边框、克制品牌色和清晰排版，不使用大面积渐变或独立营销卡片样式。它展示“章节练习 · 题量”、题目摘要、自动保存说明与“打开练习”，不在正文内嵌答案框或完整工具栏。

任务弹窗不嵌入第二个 AI 对话；需要 AI 老师时继续走已有 `askTeacher` 事件与全局 AI 表面。

## 5. 可访问性与失败边界

- 预览块使用原生按钮并声明 `aria-haspopup="dialog"`。
- 弹窗使用 `role="dialog"`、`aria-modal="true"` 和本地化名称。
- 来源元素不可见、尺寸无效或动画 API 不可用时，直接降级为淡入淡出，不阻止练习打开和关闭。
- 动画期间防止重复关闭；卸载时恢复 `body` 滚动和打开前焦点。
