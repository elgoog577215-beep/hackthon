# 变更：建立同源教学表达编译与派生同步

## Why

当前课程正文已经逐步收束到 `CourseDocument / CourseBlock`，但大纲、教案、PPT、讲义、题目导出、图解和动画仍没有正式的来源绑定、修订向量和派生依赖。继续按文件分别生成会再次形成多套语义真源，也无法在课程变化后判断哪些产物已经过期、哪些单元需要重建，以及 PPT 修改是否改变了课程含义。

## What Changes

1. 建立 `RepresentationPlan / TeachingRepresentation / RepresentationSet` 和结构化表达规格，使多种教学表达引用同一课程语义。
2. 建立 `SourceBinding / RevisionVector / AssetDerivationGraph`，记录课程对象、表达规格和产物之间的派生关系。
3. 课程正式命令持久化修订变化事件；派生系统据此精准标记过期并安排可恢复的增量重建。
4. 区分表现修改、等义修改、语义修改和无法判断的修改；语义修改进入教师/维护者的 `CourseAuthoringChange`，不得直接反写课程。
5. 建立各表达质量门、跨产物一致性检查、降级链和真实状态展示。
6. 与 `build-structured-adaptive-course-ai` 只共享稳定课程/知识引用、修订、任务和审计底座；两者分别维护业务状态机与完成标准。
7. 将 PPT 从通用“教学资源”覆盖层中拆出，建立课程级独立全屏工作台；课程顶栏提供一级入口，工作台提供同步、AI 检查、课堂演示和右上角 `.pptx` 导出。

## Out of Scope

- 本变更不允许派生产物成为课程语义真源。
- 第一阶段不要求自由视频、数字人或任意外部 PPT 的无约束回导。
- 不用高成本媒体生成阻断基础课程发布。
- 不改变公共课程、其他学生课程或跨课程正式学科库。

## Success Criteria

- 每种正式教学表达都能定位来源课程对象和源修订。
- 课程变化后，系统能确定性标记真正受影响的表达单元，不全量重建无关内容。
- 旧产物在重建失败时仍可访问，但明确标记为旧修订。
- PPT 表现修改不污染课程；语义修改必须进入统一变化候选和用户确认。
- 大纲、教案、PPT、讲义和题目通过跨产物一致性检查。
- 教师从课程顶栏一键进入独立 PPT 工作台，看到完整页序、同源依据、页级编辑、讲者备注和质量状态，并可直接全屏授课或导出中文正常、可编辑的 `.pptx`。
- 独立通过结构化同源演示与故障矩阵；再通过“基础课程更新后个人覆盖重定位”的边界协作验收。
