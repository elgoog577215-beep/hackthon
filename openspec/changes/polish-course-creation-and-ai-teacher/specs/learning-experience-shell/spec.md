## MODIFIED Requirements

### Requirement: 视觉系统必须继承既有产品语义并使用单一 token

新旧组件 MUST 使用同一色彩、圆角、边框、阴影、字号和间距 token。课程创建 MUST 继承 3 月版纵向难度和 2×2 教学风格的视觉层级，并将新增课程策略置于第二层。页面结构主要依靠背景、边框和留白，普通工作区 MUST NOT 使用玻璃光晕、装饰性大渐变、默认超大圆角或卡片套卡片。

#### Scenario: 用户配置课程生成参数

- **WHEN** 用户打开课程创建弹窗
- **THEN** 课程主题、难度和教学风格 MUST 构成清晰的第一阅读路径
- **AND** 教学结构、课程目的和资料边界 MUST 保留但不得与难度形成拥挤的左右控制台

#### Scenario: 新任务入口嵌入旧正文

- **WHEN** 任务入口出现在课程块之间
- **THEN** 它 MUST 使用与正文一致的宽度、边框、字号和状态色
- **AND** MUST NOT 看起来像来自另一套产品的独立大卡片
