## ADDED Requirements

### Requirement: 课程目录与知识蓝图必须分阶段生成

系统 MUST 先生成只包含课程定位、学习目标、章节、小节、每节唯一责任、前置小节
和范围边界的轻量目录。系统 MUST 在目录修订冻结后，按小节分别生成概念组、原子
知识点、能力、易错与掌握标准，再让模型按规范知识名称提交逐节关系候选，由系统
分配稳定知识 ID、解析端点并编译正式关系。系统 MUST NOT 要求一次模型输出同时
承载整课目录、全部知识能力包和全课关系网，也 MUST NOT 让模型创建内部知识 ID。

#### Scenario: 用户要求一章两个递进小节

- **WHEN** 用户原始要求明确指定一章和两个小节
- **THEN** 轻量目录 MUST 恰好包含一章和两个小节
- **AND** 数量验收结果 MUST 写入目录修订与最终质量报告
- **AND** 后续知识包生成 MUST NOT 增加、删除或重排小节

#### Scenario: 一个小节知识包不合法

- **WHEN** 某个小节的知识包不可解析或没有通过原子性检查
- **THEN** 系统 MUST 只纠正该小节知识包
- **AND** 已通过检查的小节目录和知识包 MUST 保持不变
- **AND** 局部纠正仍失败时 MUST 保留检查点并指出具体小节与问题

### Requirement: 每个生成阶段必须拥有独立产物与最小返工边界

同一 `GenerationJob` MUST 持久化课程 brief、目录修订、逐节知识包、关系批次、
逐节正文草稿和质量报告。每个阶段 MUST 定义输入、结构化输出、确定性检查、
检查点和最小重试单元。模型或服务失败后，恢复 MUST 从最小未完成单元继续，
MUST NOT 丢弃已经通过检查的阶段产物。

#### Scenario: 服务在第五个小节知识包生成时重启

- **WHEN** 前四个小节知识包已经通过检查并保存检查点
- **THEN** 服务恢复后 MUST 从第五个小节继续
- **AND** 前四个知识包 MUST 保持原修订和稳定知识身份

#### Scenario: 关系端点无法解析

- **WHEN** 一个关系批次引用了当前课程中不存在的知识点
- **THEN** 系统 MUST 拒绝该关系候选并只纠正目标关系批次
- **AND** 已冻结的知识结构和系统分配的稳定知识 ID MUST 保持不变
- **AND** MUST NOT 重新生成知识点、目录或正文

### Requirement: 用户审阅必须发生在昂贵知识生成之前

审阅模式 MUST 在轻量目录通过确定性检查后暂停。用户只审阅课程定位、章节、
小节、每节责任、学习目标和范围；内部知识 ID、关系端点、质量矩阵和格式修复
MUST 由系统负责。快速模式 MUST 复用相同目录检查并自动冻结目录修订。

#### Scenario: 审阅模式等待确认

- **WHEN** 轻量目录已经通过形状和责任检查
- **THEN** 任务 MUST 进入 `waiting_for_review`
- **AND** 逐节知识包、知识关系与正文生成 MUST 尚未开始
- **AND** 用户确认后 MUST 在同一 job_id 内从目录修订继续

### Requirement: 首次生成必须先编译全课一致性契约

课程目录、逐节知识包和 `CourseKnowledgeBase` 确定后，系统 MUST 编译版本化的全课一致性契约。契约 MUST 至少记录有序小节、每节唯一学习责任、硬前置、上下文承接、后续保留边界、知识与能力产出和规范术语。

#### Scenario: 蓝图包含可并行与有依赖的小节

- **WHEN** 课程蓝图完成学科、难度、资料和知识结构校验
- **THEN** 系统 MUST 生成一份与当前蓝图及课程知识库修订绑定的一致性契约
- **AND** 只有真实硬前置才能影响调度顺序
- **AND** 只需上下文衔接的小节 MUST NOT 被伪造为硬依赖

### Requirement: 节点生成必须消费最小必要的全课上下文

每个小节的正文与修复 prompt MUST 获得当前位置、前置产出、本节唯一推进、后续保留内容和规范术语。系统 MUST 允许必要的短回顾，MUST NOT 要求当前节复制前节实质讲解，也 MUST NOT 在当前节提前完成后续小节的主要任务。

#### Scenario: 多个小节并行生成

- **WHEN** 调度器同时生成没有硬依赖的多个小节
- **THEN** 每个小节 MUST 从同一份稳定契约读取自己的责任与边界
- **AND** 一致性 MUST NOT 依赖于其他并行小节先完成正文

### Requirement: 最终质量门必须检查跨小节一致性

首次发布前，系统 MUST 确定性检查小节顺序、依赖有效性、学习责任完整性、显式承接和跨节实质段落重复。短回顾和必要术语重复 MUST NOT 被当作实质正文复制。

#### Scenario: 两节包含近乎相同的长段讲解

- **WHEN** 两个小节出现超过实质段落阈值且高度相似的讲解
- **THEN** 质量门 MUST 记录定位到目标节点的阻断问题
- **AND** 课程 MUST NOT 直接发布

#### Scenario: 后一节用一两句回顾前置

- **WHEN** 当前节仅使用简短术语或承接句建立与前置的关系
- **THEN** 质量门 MUST 允许该内容
- **AND** MUST NOT 为了消除重复而删除必要的学习承接

### Requirement: 自动修复必须定点且可验收

一致性质量门 MAY 对可修复的阻断问题调用一次定点修复，但 MUST 保留未命中小节不变。修复结果只能在目标节点质量通过、原问题消失且全课阻断数下降时接受。

#### Scenario: 模型返回了新正文但没有改善一致性

- **WHEN** 修复候选仍包含原阻断问题、引入新节点问题或未通过节点质量检查
- **THEN** 系统 MUST 拒绝该候选
- **AND** MUST 保留修复前的课程工作区

### Requirement: 契约必须兼容正式课程文档与版本变化

一致性检查 MUST 同时支持任务工作区草稿和正式 `CourseDocument` 投影。当小节顺序或课程知识库修订变化时，旧契约 MUST 重新编译，MUST NOT 继续作为当前生成上下文。

#### Scenario: 学习资产编译更新课程知识库

- **WHEN** 最终学习资产编译产生新的课程知识库修订
- **THEN** 系统 MUST 在最终质量检查前重新签发一致性契约
- **AND** 新契约 MUST 引用最终课程知识库修订
